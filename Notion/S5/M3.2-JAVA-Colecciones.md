## 🧠 **1. ¿Qué es una colección en Java?**

En Java, una **colección** es una estructura que permite **almacenar, agrupar y manipular múltiples datos** de forma eficiente.

Hasta ahora hemos trabajado con **variables individuales**:

```java
int edad = 20;
```

Pero en la vida real necesitamos manejar:

- muchos trabajadores
- muchos productos
- muchos usuarios
- muchos valores

Ahí aparecen las **colecciones**.

---

## ❌ Problema sin colecciones

```java
String nombre1;
String nombre2;
String nombre3;
```

❌ No escala

❌ Difícil de mantener

❌ Poco flexible

---

## ✅ Solución con colecciones

```java
String[] nombres = new String[3];
```

O mejor aún (más adelante):

```java
List<String> nombres = new ArrayList<>();
```

---

## 🧩 **2. Tipos de Colecciones en Java**

Java ofrece varias formas de agrupar datos, cada una con un propósito distinto.

| Tipo | ¿Qué es? | Ejemplo mental |
| --- | --- | --- |
| **Array** | Tamaño fijo | Lista cerrada |
| **List** | Ordenada y flexible | Lista editable |
| **Set** | Sin duplicados | Conjunto único |
| **Map** | Clave → valor | Diccionario |
| **Stream** | Procesamiento | Filtro de datos |

---

## 🗂️ **3. Comparación general**

| Característica | Array | List | Set | Map |
| --- | --- | --- | --- | --- |
| Tamaño fijo | ✔ | ❌ | ❌ | ❌ |
| Permite duplicados | ✔ | ✔ | ❌ | Claves ❌ |
| Mantiene orden | ✔ | ✔ | Depende | No siempre |
| Acceso por índice | ✔ | ✔ | ❌ | ❌ |
| Clave–valor | ❌ | ❌ | ❌ | ✔ |
| Parte del Collections Framework | ❌ | ✔ | ✔ | ✔ |

## 🧭 **4. ¿Cuándo usar cada uno?**

### 🔹 Array

Usar cuando:

- Sabes cuántos elementos habrá
- El tamaño no cambia
- Estás comenzando a aprender

Ejemplo:

- 3 trabajadores
- 7 días de la semana
- 12 meses

---

### 🔹 List (`ArrayList`)

Usar cuando:

- No sabes cuántos elementos habrá
- Necesitas agregar o eliminar
- El orden importa

Ejemplo:

- Lista de alumnos
- Carrito de compras
- Historial de acciones

---

### 🔹 Set

Usar cuando:

- NO deben existir duplicados
- El orden no es lo principal

Ejemplo:

- Correos únicos
- RUT únicos
- IDs únicos

---

### 🔹 Map

Usar cuando:

- Necesitas una **clave** para buscar un valor
- El acceso debe ser rápido

Ejemplo:

- usuario → contraseña
- idProducto → precio
- rut → trabajador

---

### 🔹 Stream

Usar cuando:

- Quieres **procesar** colecciones
- Filtrar, transformar o resumir datos
- Evitar loops manuales

Ejemplo:

- Filtrar sueldos mayores a X
- Contar trabajadores
- Calcular promedios

---

## ⚙️ **5. Ejemplos simples**

### Array

```java
int[] edades = {20, 25, 30};
```

---

### List

```java
List<String> nombres = new ArrayList<>();
nombres.add("Ana");
nombres.add("Juan");
```

---

### Set

```java
Set<String> correos = new HashSet<>();
correos.add("a@mail.com");
correos.add("a@mail.com"); // duplicado → ignorado
```

---

### Map

```java
Map<String, Integer> sueldos = new HashMap<>();
sueldos.put("Juan", 500000);
```

---

### Stream

```java
nombres.stream()
       .forEach(System.out::println);
```

*(Streams se verán más adelante con calma)*

## 🧼 **6. Buenas Prácticas Generales**

✔ Elegir la colección correcta según el problema

✔ Usar `List` en lugar de arrays cuando el tamaño cambia

✔ Usar `Set` para evitar duplicados sin lógica extra

✔ Usar `Map` cuando hay relación clave–valor

✔ Leer el código pensando en **intención**, no en moda

---

## ❌ **7. Malas Prácticas Comunes**

❌ Usar arrays para todo

❌ Forzar índices cuando no corresponde

❌ Usar `List` cuando no importa el orden

❌ Usar `Map` solo porque “es más pro”

❌ Anidar loops complejos cuando un Stream sería más claro
