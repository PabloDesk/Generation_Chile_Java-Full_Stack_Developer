# JAVA - `return` en Java

## 🎯 Objetivo

Entender qué hace `return`, cuándo usarlo y cómo mejora la lógica del código.

---

# 🧠 ¿Qué es `return`? 🔙

`return` se usa para:

1. **Terminar un método inmediatamente**
2. **Devolver un valor desde un método**
3. **Salir anticipadamente de una validación**

---

# 🧩 1. `return` para salir de un método

Cuando Java encuentra `return`, **el método se detiene inmediatamente**.

```java
public class EjemploReturn {
    public static void main(String[] args) {

        System.out.println("Inicio");

        return;

        // Esto nunca se ejecuta
        // System.out.println("Fin");
    }
}
```

Resultado:

```
Inicio
```

---

# 🧩 2. `return` para validaciones (uso más común)

Esto es lo más útil para alumnos nuevos.

```java
int edad = -5;

if (edad < 0) {
    System.out.println("Edad inválida");
    return;
}

System.out.println("Edad correcta");
```

Aquí ocurre:

1. Valida error
2. Si hay error → `return`
3. El resto del código no se ejecuta

---

# 🎯 ¿Por qué usar `return`?

Evita esto:

```java
if (edad >= 0) {
    // lógica grande
}
```

Mejor esto:

```java
if (edad < 0) {
    return;
}

// lógica limpia
```

Esto se llama:

> **Early return** (salida temprana)
> 

---

# 🧩 3. Ejemplo real (Sistema Cine)

```java
if (precioBase <= 0) {
    System.out.println("Precio inválido");
    return;
}

if (edad < 0) {
    System.out.println("Edad inválida");
    return;
}
```

Ventaja:

- código más limpio
- menos anidación
- más legible

---

# 🧩 4. `return` devolviendo valores

Cuando un método devuelve algo.

```java
public static int sumar(int a, int b) {
    return a + b;
}
```

Uso:

```java
int resultado = sumar(5, 3);
System.out.println(resultado);
```

Salida:

```
8
```

---

# 🧩 5. `return` con condicionales

```java
public static String evaluarEdad(int edad) {

    if (edad < 0) {
        return "Edad inválida";
    }

    if (edad < 18) {
        return "Menor de edad";
    }

    return "Mayor de edad";
}
```

Esto es muy usado en Java moderno.

---

# ⚠️ Regla importante

Después de `return` **no se ejecuta nada más**

```java
return;
System.out.println("Nunca se ejecuta");
```

---

# 🚫 Error común

Esto es incorrecto:

```java
public static int numero() {
    System.out.println("Hola");
}
```

Debe devolver algo:

```java
public static int numero() {
    return 10;
}
```

---

# 🎯 Cuándo usar `return`

Usar `return` cuando:

✔ Validar errores

✔ Salir anticipadamente

✔ Devolver resultados

✔ Evitar if anidados

✔ Simplificar lógica

---

# 🧪 Ejercicio para alumnos

Crear un programa que:

- Pida una edad
- Si la edad es negativa → usar `return`
- Si no → evaluar mayor de edad

```java
import java.util.Scanner;

public class EjercicioReturn {

    public static void main(String[] args) {

        Scanner scanner = new Scanner(System.in);

        System.out.print("Ingrese edad: ");
        int edad = scanner.nextInt();

        if (edad < 0) {
            System.out.println("Edad inválida");
            return;
        }

        if (edad >= 18) {
            System.out.println("Mayor de edad");
        } else {
            System.out.println("Menor de edad");
        }

        scanner.close();
    }
}
```

---

# 🧠 Resumen

`return` sirve para:

- terminar un método
- devolver valores
- validar errores
- limpiar el código

---

# 🎤 Cómo explicarlo en clase

Puedes decir:

> `return` es como decirle al programa:
> 
> 
> "hasta aquí llegamos, salimos ahora mismo"
> 

---

# 🚀 Buena práctica moderna

Preferir:

```java
if (error) return;
```

En vez de:

```java
if (!error) {
    // todo el código
}
```

Esto hace el código más profesional y legible.
