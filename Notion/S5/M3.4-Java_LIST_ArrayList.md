# Java - LIST (ArrayList)

Al finalizar esta sección, el estudiante será capaz de:

- Entender qué es una `List` en Java
- Comprender por qué `List` existe y cuándo usarla
- Usar `ArrayList` correctamente
- Agregar, eliminar y acceder a elementos
- Recorrer listas con distintos loops
- Evitar errores comunes

---

## 🧠 **1. ¿Qué es una List en Java?**

Una **List** es una colección que permite almacenar **múltiples elementos del mismo tipo**, **en orden**, y cuyo **tamaño puede cambiar dinámicamente**.

En Java, la implementación más usada es:

```java
ArrayList
```

---

## ❓ ¿Por qué existe List si ya tenemos Arrays?

Porque los arrays tienen **limitaciones importantes**:

| Problema del Array | Solución con List |
| --- | --- |
| Tamaño fijo | Tamaño dinámico |
| Manejo manual de índices | Gestión automática |
| Difícil eliminar elementos | Métodos claros (`remove`) |
| Código más frágil | Código más seguro |

👉 **List aparece para simplificar la vida del desarrollador.**

---

## 🧩 **2. Jerarquía básica (sin POO profundo)**

```
List  →  ArrayList
```

- `List` → interfaz (contrato)
- `ArrayList` → implementación concreta

📌 **Siempre programamos contra `List`, no contra `ArrayList`.**

---

## ✍️ **3. Cómo declarar una List**

### ✔ Forma correcta (recomendada)

```java
List<String> nombres = new ArrayList<>();
```

---

### ❌ Forma menos recomendable

```java
ArrayList<String> nombres = new ArrayList<>();
```

📌 ¿Por qué?

- Pierdes flexibilidad
- Te acoplas a la implementación

---

## 🧱 **4. Agregar elementos**

```java
List<String> nombres = new ArrayList<>();

nombres.add("Ana");
nombres.add("Juan");
nombres.add("Pedro");

```

✔ No necesitas tamaño inicial

✔ No te preocupas por índices

---

## 🔎 **5. Acceder a elementos**

```java
System.out.println(nombres.get(0)); // Ana
```

📌 **Los índices siguen comenzando en 0**, igual que los arrays.

---

## ✏️ **6. Modificar elementos**

```java
nombres.set(1, "María");
```

---

## 🗑️ **7. Eliminar elementos**

### Por índice

```java
nombres.remove(0);
```

### Por valor

```java
nombres.remove("Pedro");
```

✔ La lista se reorganiza sola

✔ No quedan “espacios vacíos”

---

## 📏 **8. Tamaño de una List**

```java
int total = nombres.size();
```

📌 A diferencia de arrays:

- `length` ❌
- `size()` ✔

---

## 🔁 **9. Recorrer una List**

### 🔹 Con `for`

```java
for (int i = 0; i < nombres.size(); i++) {
    System.out.println(nombres.get(i));
}
```

---

### 🔹 Con `foreach` (recomendado)

```java
for (String nombre : nombres) {
    System.out.println(nombre);
}
```

✔ Más limpio

✔ Más legible

---

### 🔹 Con `while`

```java
int i = 0;
while (i < nombres.size()) {
    System.out.println(nombres.get(i));
    i++;
}
```

---

## ⚠️ **10. Errores Comunes con List**

### ❌ Acceder a índice inexistente

```java
nombres.get(10); // IndexOutOfBoundsException
```

✔ Solución:

```java
if (i < nombres.size()) { ... }
```

---

### ❌ Confundir `length` con `size()`

```java
nombres.length; // ❌
nombres.size(); // ✔
```

---

### ❌ Modificar una List dentro de un foreach

```java
for (String n : nombres) {
    nombres.remove(n); // ❌
}
```

👉 Esto lanza error en tiempo de ejecución.

---

## 🧼 **11. Buenas Prácticas con List**

✔ Usar `List` como tipo

✔ Preferir `foreach` para lectura

✔ Usar `remove()` con cuidado

✔ Verificar `size()` antes de acceder

✔ Usar `List` cuando el tamaño es variable

---

## ❌ **12. Malas Prácticas**

❌ Usar `ArrayList` directamente como tipo

❌ Usar índices mágicos

❌ Usar List cuando NO importa el orden

❌ Usar List solo “porque es más moderna”

---

## 🔄 **13. Comparación: Array vs List**

| Característica | Array | List |
| --- | --- | --- |
| Tamaño | Fijo | Dinámico |
| Agregar elementos | ❌ | ✔ |
| Eliminar elementos | ❌ | ✔ |
| Sintaxis | Simple | Más rica |
| Uso recomendado | Datos fijos | Datos variables |

---

## 🧠 **14. Ejemplo aplicado (Payroll)**

### Antes (Array):

```java
String[] nombres = new String[3];
```

### Después (List):

```java
List<String> nombres = new ArrayList<>();
```

👉 Ya no necesitas:

- contadores manuales
- `null`
- límites artificiales

> [!CAUTION]

>
> **Array se queda corto** y
>
> **List toma el control**.

---

## ListExample.java

```java
import java.util.ArrayList;
import java.util.List;

public class ListExample {

    public static void main(String[] args) {

        //TAMAÑO DINAMICO
        // ESTE SI TIENE AGREGAR ... ADD, REMOVE, FOREACH

        List<String> trabajadores = new ArrayList<>();

        //AGREGA
        trabajadores.add("Juan");
        trabajadores.add("Pedro");
        trabajadores.add("Diego");

        // ELIMINA
        trabajadores.remove("Juan");
        System.out.println(" = LIST =");

        for (String t: trabajadores){
            System.out.println(t);
        }

        System.out.println("Total :" + trabajadores.size());
    }
}
```
