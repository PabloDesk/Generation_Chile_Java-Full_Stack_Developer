# 🧪  Propuesta de Actividad: AdvancedCalculator 🎯

## 📌 Descripción general

En esta actividad, cada equipo deberá desarrollar una aplicación de consola llamada:

```markdown
AdvancedCalculator.java
```

El objetivo es construir una **calculadora avanzada** que permita realizar distintas operaciones matemáticas usando **funciones (métodos)**, reforzando la modularización del código, la reutilización de lógica y la organización de un proyecto real en Java.

Además, el proyecto deberá subirse a **GitHub** con:

- `README.md`
- `.gitignore`
- commits ordenados
- estructura limpia del repositorio

---

# 🎓 Objetivos de aprendizaje

Al finalizar esta actividad, cada equipo habrá practicado:

✔ Uso de `Scanner` para entrada de datos

✔ Uso de `switch` o `if/else` para el menú de opciones

✔ Creación y uso de **funciones con retorno**

✔ Separación de responsabilidades en métodos

✔ Validación básica de datos

# 🧮 Enunciado

Cada grupo deberá construir una calculadora por consola que permita al usuario elegir entre varias operaciones matemáticas.

El sistema debe incluir como mínimo las siguientes opciones:

### 1. Sumar

Debe pedir dos números y mostrar el resultado.

### 2. Restar

Debe pedir dos números y mostrar el resultado.

### 3. Multiplicar

Debe pedir dos números y mostrar el resultado.

### 4. Dividir

Debe pedir dos números y validar que el divisor no sea `0`.

### 5. Potencia

Debe calcular un número elevado a otro.

Ejemplo:

- `2^3 = 8`

### 6. Raíz cuadrada

Debe calcular la raíz cuadrada de un número.

Validar que el número no sea negativo.

### 7. Módulo o residuo

Debe mostrar el residuo de una división.

Ejemplo:

- `10 % 3 = 1`

### 8. Salir

Debe cerrar el programa.

---

# 🧩 Requerimientos técnicos obligatorios

## 🔹 Debe usar funciones sí o sí

Cada operación debe estar en su propio método.

Ejemplo esperado:

```java
static double sumar(double a, double b)
static double restar(double a, double b)
static double multiplicar(double a, double b)
static double dividir(double a, double b)
static double potencia(double base, double exponente)
static double raizCuadrada(double numero)
static double modulo(double a, double b)
```

También puede existir una función para mostrar el menú:

```java
static void mostrarMenu()
```

---

## 🔹 Debe usar `Scanner`

Se debe utilizar para:

- leer la opción del usuario
- leer los números necesarios para cada operación

---

## 🔹 Debe tener control de flujo

El programa debe:

- mostrar un menú repetitivo
- permitir volver a operar varias veces
- salir solo cuando el usuario elija la opción de salir

Se recomienda usar:

- `do-while` para el menú principal
- `switch` para procesar opciones

---

## 🔹 Debe validar casos importantes

Como mínimo:

- no dividir por cero
- no sacar raíz cuadrada de números negativos
- no hacer módulo con divisor cero
- mostrar mensaje cuando la opción sea inválida

---

## 🔹 Debe estar modularizado

No se debe hacer toda la lógica dentro de `main`.

El `main` debe quedar principalmente para:

- crear el `Scanner`
- manejar el menú
- pedir datos
- llamar funciones
