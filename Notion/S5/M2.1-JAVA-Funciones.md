# JAVA - Funciones

# 🧠 **1. ¿Qué es una Función (Método) en Java?**

Una función (en Java llamada **método**) es un **bloque de código reutilizable** que ejecuta una tarea específica.

> [!IMPORTANT]
> Una función es como darle un nombre a un conjunto de instrucciones.
> 
> En lugar de copiar y pegar el mismo código muchas veces, lo escribimos una sola vez y lo reutilizamos.

---

## ❌ Problema SIN funciones

```java
System.out.println("Bienvenido");
System.out.println("1. Jugar");
System.out.println("2. Salir");

// … repetir lo mismo en varias partes del código
```

---

## ✅ Solución CON funciones

```java
mostrarMenu();
```

---

# 🧩 **2. ¿Por qué necesitamos funciones?**

Las funciones nos permiten:

✔ Evitar código repetido

✔ Hacer programas más **legibles**

✔ Dividir problemas grandes en partes pequeñas

✔ Facilitar mantenimiento

✔ Preparar el código para proyectos reales

> [!TIP]
>  Si tu método main tiene más de 50–60 líneas, probablemente necesitas funciones.

---

# 🧱 **3. Estructura de una Función en Java**

```java
modificador tipoRetorno nombreFuncion(parametros) {
    // código
    return valor; // solo si retorna algo
}
```

Ejemplo:

```java
static void saludar() {
    System.out.println("Hola desde una función");
}
```

---

## 🧩 Partes explicadas

| Parte | Significado |
| --- | --- |
| `static` | Permite usar el método sin crear objetos (por ahora obligatorio) |
| `void` | No retorna nada |
| `saludar` | Nombre del método |
| `()` | Parámetros (vacío en este caso) |

---

# 🟦 **4. Funciones SIN Retorno (`void`)**

Se usan cuando la función **solo ejecuta acciones**.

---

### ✔ Ejemplo básico

```java
static void mostrarMenu() {
    System.out.println("1. Ver perfil");
    System.out.println("2. Entrenar");
    System.out.println("3. Salir");
}
```

Uso:

```java
mostrarMenu();
```

---

### ✔ Ejemplo con parámetros

```java
static void mostrarMensaje(String mensaje) {
    System.out.println(mensaje);
}
```

Uso:

```java
mostrarMensaje("Bienvenido al sistema");
```

---

# 🟩 **5. Funciones CON Retorno**

Se usan cuando la función **calcula y devuelve un valor**.

---

### ✔ Ejemplo simple

```java
static int sumar(int a, int b) {
    return a + b;
}
```

Uso:

```java
int resultado = sumar(5, 3);
System.out.println(resultado);
```

---

### ✔ Ejemplo con lógica

```java
static boolean esMayorDeEdad(int edad) {
    return edad >= 18;
}
```

Uso:

```java
if (esMayorDeEdad(20)) {
    System.out.println("Acceso permitido");
}
```

---

# 🧮 **6. Parámetros y Argumentos**

- **Parámetros** → lo que recibe la función
- **Argumentos** → lo que se envía al llamarla

```java
static int multiplicar(int x, int y) {
    return x * y;
}

int total = multiplicar(4, 5);
```

# 🔁 **7. Funciones + Control de Flujo**

Las funciones pueden usar:

- `if / else`
- `switch`
- `loops`

---

### ✔ Ejemplo realista (menú)

```java
static void procesarOpcion(int opcion) {
    switch (opcion) {
        case 1 -> System.out.println("Perfil");
        case 2 -> System.out.println("Entrenamiento");
        case 3 -> System.out.println("Salir");
        default -> System.out.println("Opción inválida");
    }
}
```

---

# 🔄 **8. Funciones + Loops**

```java
static void cuentaRegresiva(int inicio) {
    for (int i = inicio; i >= 1; i--) {
        System.out.println(i);
    }
}
```

# ⚠️ **9. Reglas IMPORTANTES**

### 🔴 Regla 1

**Todas las funciones deben ser `static` por `ahora`.**

### 🔴 Regla 2

Las funciones **NO pueden declararse dentro del `main`**.

❌ Incorrecto:

```java
public static void main(String[] args) {
    static void saludar() { } // ❌
}
```

---

### 🔴 Regla 3

Una función debe hacer **UNA sola cosa**.

❌ Mal ejemplo:

```java
static void hacerTodo() {
    // menú
    // cálculos
    // impresión
    // validaciones
}

```

---

# 🧼 **10. Buenas Prácticas con Funciones**

✔ Nombres claros y verbales

`mostrarMenu()`, `calcularTotal()`, `validarEdad()`

✔ Métodos cortos (10–20 líneas idealmente)

✔ Reutilización

Si copias código → probablemente necesita función

✔ Ordenar funciones debajo del `main`

---

# ❌ **11. Malas Prácticas Comunes**

❌ Funciones gigantes

❌ Usar nombres genéricos (`metodo1`, `hacerCosas`)

❌ No usar retorno cuando corresponde

❌ Retornar valores y no usarlos

❌ Abusar de parámetros innecesarios

---

# 🎮 **12. Ejemplos Didácticos Completos**

### 🟦 Ejemplo 1 - Menú organizado con funciones

```java
static void mostrarMenu() {
    System.out.println("1. Jugar");
    System.out.println("2. Configuración");
    System.out.println("3. Salir");
}
```

---

### 🟩 Ejemplo 2 - Lógica separada

```java
static int calcularDaño(int ataque, int defensa) {
    int daño = ataque - defensa;
    return daño > 0 ? daño : 0;
}
```

---

### 🟥 Ejemplo 3 - Función booleana

```java
static boolean tieneEnergia(int energia) {
    return energia > 0;
}
```
