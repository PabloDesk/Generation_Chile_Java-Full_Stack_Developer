# [CONTRIBUTING.md](http://contributing.md/) 🤝 

# 📌 ¿Qué es este archivo?

Este documento define **cómo contribuir al repositorio**.

Todos los alumnos deben seguir estas reglas antes de subir código.

---

# 🎯 Objetivo

- Mantener orden del proyecto
- Evitar conflictos
- Definir flujo de trabajo
- Simular entorno profesional

---

# 🌿 Flujo de trabajo

1. Actualizar repositorio

```
git pull origin main
```

1. Crear rama

```
git checkout -b feature/nombre-tarea
```

1. Trabajar normalmente
2. Agregar cambios

```
git add .
```

1. Commit

```
git commit -m "feat: descripcion"
```

1. Subir rama

```
git push origin feature/nombre-tarea
```

1. Crear Pull Request

---

# 🚨 Reglas

- ❌ No trabajar en main
- ❌ No hacer push directo a main
- ✔ Usar ramas
- ✔ Crear Pull Request
- ✔ Commits claros

---

# 🧾 Convención de commits

Formato:

```
tipo: descripcion
```

Tipos:

- feat → nueva funcionalidad
- fix → corrección
- docs → documentación
- style → estilos
- refactor → mejora código
- test → pruebas

Ejemplos:

```
feat: agregar calculadora
fix: corregir validacion edad
docs: actualizar readme
```

---

# 🌱 Convención de ramas

Formato:

```
tipo/nombre
```

Ejemplos:

```
feature/login
feature/calculadora
fix/error-menu
docs/readme
```

---

# 👥 Reglas de equipo

Siempre:

1. Crear rama
2. Subir cambios
3. Crear Pull Request
4. Revisión
5. Merge

---

# ✅ Buenas prácticas

- Código limpio
- No subir código roto
- Probar antes de commit
- Commits claros
- Respetar estructura

---

# 🏁 Regla final
Nunca subir directo a main.
Siempre trabajar con ramas.
