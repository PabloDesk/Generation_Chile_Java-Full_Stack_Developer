# JAVA - Bucles

# 🌀 **1. ¿Qué es un Bucle en Java?**

Un **bucle** (o *loop*) es una estructura que permite **repetir un bloque de código múltiples veces**, mientras se cumpla una condición.

Los loops son fundamentales porque permiten:

- Automatizar tareas repetitivas
- Procesar datos de manera eficiente
- Simular ciclos, turnos, procesos o iteraciones
- Construir juegos, menús, validaciones y más

**Sin loops** tendríamos que copiar y pegar el mismo código muchas veces.

---

# 🎯 **2. Tipos de Bucles en Java**

Java incluye cuatro tipos principales de bucles:

| Tipo de Bucle | Cuándo se usa | Característica principal |
| --- | --- | --- |
| **while** | Cuando no sabes cuántas veces se repetirá | Repite mientras una condición sea verdadera |
| **do-while** | Cuando debe ejecutarse *al menos una vez* | Evalúa la condición después |
| **for** | Cuando sabes cuántas iteraciones necesitas | Control total del contador |
| **for-each** | Cuando recorres arreglos o colecciones | Más legible y seguro |

*(For-each se enseña formalmente cuando lleguemos a arrays, pero lo incluimos para el módulo de loops.)*

---

# 🧩 **3. Bucle WHILE**

El **while** repite un bloque siempre que la condición sea verdadera.

## 📘 Sintaxis:

```java
while (condicion) {
    // código a repetir
}
```

## ✔ Ejemplo básico:

```java
int contador = 1;

while (contador <= 5) {
    System.out.println("Iteración " + contador);
    contador++; // evita loop infinito
}
```

---

# 🧠 ¿Cuándo usar while?

- Si NO sabemos cuántas veces se repetirá.
- Cuando la repetición depende del usuario o de la lógica del programa.
- En validaciones, menús, intentos o simulaciones.

---

# ⚠️ Errores comunes con while

❌ Loop infinito:

```java
while (i < 5) {
    System.out.println(i);
}
// i nunca cambia
```

---

# 🧩 **4. Bucle DO-WHILE**

Se ejecuta **al menos una vez**, porque la condición se evalúa al final.

## 📘 Sintaxis:

```java
do {
    // código a ejecutar
} while (condicion);
```

## ✔ Ejemplo:

```java
int numero = 0;

do {
    System.out.println("Esto se ejecuta una vez como mínimo");
    numero++;
} while (numero < 0);
```

Aunque la condición es falsa, el bloque se ejecuta una vez.

---

# 🧠 ¿Cuándo usar do-while?

- Menús interactivos
- Confirmaciones
- Situaciones donde el usuario debe ver algo primero
- Procesos que siempre deben correr al menos una vez

---

# 🧩 **5. Bucle FOR**

Ideal cuando conoces la cantidad de repeticiones.

## 📘 Sintaxis:

```java
for (inicial; condicion; incremento) {
    // código repetido
}
```

## ✔ Ejemplo:

```java
for (int i = 1; i <= 10; i++) {
    System.out.println("Número: " + i);
}
```

---

# 🧠 ¿Cuándo usar for?

- Contadores
- Rango de números
- Simulaciones con pasos fijos
- Procesos con iteración conocida (ej: 10 turnos, 5 días, 3 intentos)

---

# 🧩 **6. Bucle FOR-EACH**

*(Relacionado a arreglos y colecciones, pero útil en loops)*

Sirve para recorrer arreglos de forma más simple y segura.

## ✔ Ejemplo:

```java
String[] frutas = {"Manzana", "Pera", "Naranja"};

for (String f : frutas) {
    System.out.println(f);
}
```

---

# 🔥 **7. Comparación de Bucles**

| Bucle | Mejor uso | ¿Evalúa antes o después? |
| --- | --- | --- |
| **while** | Repetición desconocida | Antes |
| **do-while** | Ejecutar al menos una vez | Después |
| **for** | Repetición exacta | Antes |
| **foreach** | Recorrer elementos | Antes |

---

# 🧼 **8. Buenas Prácticas al Usar Bucles**

### ✔ Claridad en la condición

Evita condiciones demasiado largas o poco claras.

### ✔ Incremento dentro del loop

Evita loops infinitos.

### ✔ Nombres descriptivos para contadores cuando sea necesario

❌ `int x = 0;`

✔ `int turno = 0;`

### ✔ Evitar modificar el contador dentro del cuerpo del loop (excepto cuando es intencional)

### ✔ Evitar múltiples responsabilidades dentro del loop

Cada loop debe hacer *una cosa*.

### ✔ Revisar límites cuidadosamente

Errores comunes: `<=`, `<`, `>=`, `>`.

---

# 🚫 **9. Malas Prácticas Comunes**

❌ Usar un while para un número fijo de repeticiones.

❌ No actualizar la variable de control.

❌ Anidar demasiados loops sin necesidad.

❌ Usar `break` sin justificación clara.

❌ Usar nombres como `i`, `j`, `k` en loops no triviales.

# 🎮 **10. Ejemplos Reales y Claros**

## 🟦 Ejemplo 1 — Contador simple

```java
for (int i = 1; i <= 5; i++) {
    System.out.println("Vueltas: " + i);
}
```

---

## 🟩 Ejemplo 2 — Simulación de vida

```java
int vida = 50;

while (vida > 0) {
    System.out.println("Vida actual: " + vida);
    vida -= 10;
}

System.out.println("Personaje derrotado.");
```

---

## 🟧 Ejemplo 3 — Menú con do-while

```java
int opcion;

do {
    System.out.println("1. Jugar");
    System.out.println("2. Salir");
    opcion = scanner.nextInt();
} while (opcion != 2);
```

---

## 🟥 Ejemplo 4 — Tabla de multiplicar

```java
int n = 7;

for (int i = 1; i <= 10; i++) {
    System.out.println(n + " x " + i + " = " + (n * i));
}
```
