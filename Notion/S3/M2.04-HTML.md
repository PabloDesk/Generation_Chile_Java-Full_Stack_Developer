## 🧪 ¿Qué es realmente una página web?

Imagina que escribes en un Bloc de Notas:

```
hola esta es mi pagina web
```

Si lo guardas como `miweb.html`… el navegador igual puede mostrarlo.

¿Por qué?

Porque **una página web es simplemente contenido**, y el navegador sabe mostrar texto por sí solo.

Entonces, **¿para qué usar HTML?**

👉 Para darle **estructura**, **orden**, **significado** y **accesibilidad** al contenido.

HTML permite decirle al navegador:

- qué es un título
- qué es un párrafo
- qué es una imagen
- qué es una tabla
- qué parte es el header, el footer, etc.

<aside>

> Como developers, nuestra responsabilidad es escribir HTML bien estructurado, semántico y accesible.
> 
</aside>

## ✨ Qué es HTML (y qué no es)

HTML **no es un lenguaje de programación**.

No tiene lógica, no tiene variables, no tiene `if`, no tiene bucles.

Es un **lenguaje de marcado**.

Asignamos etiquetas para describir los elementos de una página.

Ejemplo:

```html
<h1>Hola Mundo</h1>
<p>Este es mi primer sitio web.</p>
```

### 🏷 ¿Qué es una etiqueta?

Una “instrucción visual” para el navegador.

Las etiquetas pueden ser:

- **Normales** → tienen apertura y cierre
- **Vacías / self-closing** → no envuelven contenido

Ejemplos self-closing:

```html
<img src="foto.jpg" alt="Foto" />
<br />
<hr />
```

---

## 🧱 ¿Cómo funciona HTML?

HTML forma un **árbol de elementos**:

padres → hijos → hermanos → ramas.

Esto determina:

- qué se muestra primero
- qué está dentro de qué
- cómo el CSS/JS trabajará luego con esa estructura

---

# 📖 Estructura base de un documento HTML5

```html
<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Mi Primera Web</title>
  </head>
  <body>
    <h1>Bienvenidos</h1>
    <p>Este es el inicio de mi portafolio web.</p>
  </body>
</html>
```

## 🔽 **7. Elementos semánticos modernos (HTML5 real)**

```html
<header>Encabezado del sitio</header>

<nav>
  <a href="#">Inicio</a> |
  <a href="#">Proyectos</a>
</nav>

<main>
  <article>
    <h3>Artículo destacado</h3>
    <p>Contenido del artículo.</p>
  </article>

  <section>
    <h3>Sección</h3>
    <p>Contenidos generales.</p>
  </section>
</main>

<aside>
  Información adicional
</aside>

<footer>
  © 2025 Mi Sitio
</footer>
```

---

## 🔽 **8. Div y Span (bloque vs en línea)**

```html
<div style="background:#222; color:white;">
  Este es un DIV (ocupa todo el ancho)
</div>

<p>
  Texto con un <span style="color:red;">SPAN (en línea)</span> dentro.
</p>
```

# 💡 Tip avanzado: ¿Por qué es importante el `index.html`?

Cuando subes un sitio web, el servidor busca automáticamente `index.html` como página inicial.

Es el “punto de entrada” de la web.

Por eso siempre:

```
index.html
```

# 🎉 **¿Qué puedes construir solo con HTML?**

- Páginas personales
- Blogs simples
- Formularios completos
- Documentación
- Estructura para landing pages
- Prototipos de apps
- Mockups sin CSS
- Contenido para emails
- Plantillas para proyectos

> [!TIP]
> Cuando sumes CSS + JS, esto se convierte en sitios completos.
