# Functions & Algorithms

## 🎯 Objetivo de la clase

Que el estudiante entienda:

- Qué es una **función**
- Qué es un **algoritmo**
- Cómo se relacionan
- Por qué son clave para ser developer

---

## 🧠 1. ¿Qué es una Function?

👉 Una **función** es un bloque de código que:

- Tiene un propósito específico
- Puede recibir datos (inputs)
- Puede devolver un resultado (output)

### 📌 Ejemplo en JavaScript

```jsx
function saludar(nombre) {
  return "Hola " + nombre;
}

console.log(saludar("Hackerman"));
```

### 🧩 Traducción simple

> [!NOTE]
> Una función es como una **máquina**:
> 
> - Le das algo → lo procesa → te devuelve algo

---

## 🧠 2. ¿Por qué usamos funciones?

### 💡 Problemas que resuelven:

- Evitan repetir código (DRY)
- Hacen el código más ordenado
- Permiten dividir problemas grandes

### 📌 Ejemplo real

❌ Sin función:

```jsx
console.log("Hola Juan");
console.log("Hola María");
console.log("Hola Pedro");
```

✅ Con función:

```jsx
function saludar(nombre) {
  return "Hola " + nombre;
}

saludar("Juan");
saludar("María");
saludar("Pedro");
```

---

## 🧠 3. ¿Qué es un Algorithm?

👉 Un **algoritmo** es:

> Una serie de pasos ordenados para resolver un problema
> 

### 📌 Ejemplo cotidiano

**Hacer un café ☕**

1. Hervir agua
2. Agregar café
3. Mezclar
4. Servir

➡️ Eso es un algoritmo

---

## 💻 Ejemplo en código

```jsx
function hacerCafe() {
  console.log("Hervir agua");
  console.log("Agregar café");
  console.log("Mezclar");
  console.log("Servir");
}
```

---

## 🔗 4. Relación clave (IMPORTANTÍSIMO)

👉 Un algoritmo = la lógica

👉 Una función = cómo implementas esa lógica en código

💡 En simple:

> [!TIP]
> **El algoritmo es la idea, la función es la ejecución**

---

## 🧠 5. Ejemplo completo

### 🎯 Problema:

Calcular si un número es par

### 🧩 Algoritmo:

1. Recibir número
2. Dividir por 2
3. Verificar si el resto es 0
4. Retornar resultado

### 💻 Función:

```jsx
function esPar(numero) {
  return numero % 2 === 0;
}

console.log(esPar(4)); // true
console.log(esPar(5)); // false
```

---

## ⚠️ 6. Errores comunes

🚫 Pensar que función = algoritmo

🚫 No separar lógica en pasos

🚫 Funciones que hacen demasiadas cosas

🚫 No reutilizar funciones

---

## 🧪 7. Mini actividad  Ejemplo

### 🎯 Problema:

Crear una función que diga si alguien puede entrar a una discoteca

### 📌 Reglas:

- Si tiene 18 o más → puede entrar
- Si no → no puede

### 🧩 Paso 1: Algoritmo (en palabras)

👉 (los alumnos lo escriben)

### 💻 Paso 2: Código

```jsx
function puedeEntrar(edad) {
  if (edad >= 18) {
    return "Puede entrar";
  } else {
    return "No puede entrar";
  }
}
```

---

## 🧠 8. Mentalidad clave (lo más importante)

> [!NOTE]
> Un developer no solo escribe código…
> 
> **resuelve problemas paso a paso**

---

## 🧠 Resumen final

- Función = bloque reutilizable de código
- Algoritmo = pasos para resolver un problema
- Primero piensas → luego programas
- Buen código = buenas decisiones + buena lógica
