# Actividad: Crear mi CV solo con HTML

> [!NOTE]
> **Objetivo:** construir un currículum vitae utilizando únicamente **HTML**, sin estilos CSS, para practicar estructura, etiquetas semánticas, jerarquía de información y buenas decisiones de marcado.
>

---

## 🎯 Objetivos de aprendizaje

Al finalizar esta actividad, el estudiante debería poder:

- comprender que **HTML estructura** el contenido, no lo “decora”
- diferenciar entre títulos, párrafos, listas, enlaces, imágenes y tablas
- usar etiquetas HTML de forma intencional según el contenido
- organizar información personal y profesional dentro de una página web
- construir un documento HTML básico, ordenado y entendible

---

## 🧠 Idea central de la actividad

En esta actividad **no importa que el CV se vea bonito**.

Lo importante es que esté:

- bien estructurado
- ordenado
- completo
- usando la **etiqueta correcta para cada parte**

La meta es que el navegador pueda leer correctamente el contenido y que ustedes entiendan **por qué existe cada etiqueta HTML**.

---

## 🚫 Restricciones obligatorias

- **No se permite CSS**
- **No se permite JavaScript**
- **No se permite copiar una plantilla completa desde internet**
- Deben escribir el archivo manualmente
- El archivo principal debe llamarse:

```
index.html
```

---

## 📁 Entregable esperado

Cada estudiante debe entregar:

- un archivo `index.html`
- su CV en formato página web
- opcionalmente una imagen de perfil si desea usar `<img>`

---

# 🏗 Estructura obligatoria del CV

Deben construir su CV con las siguientes secciones.

---

## 1. Encabezado del documento

Todo CV debe comenzar con la estructura base de HTML5:

```html
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>CV de Tu Nombre</title>
</head>
<body>
</body>
</html>
```

---

## 2. Encabezado principal del CV

### Deben usar:

- `<header>` para el bloque principal
- `<h1>` para el nombre completo
- `<p>` para una breve descripción profesional
- `<img>` si desean agregar foto

### Ejemplo esperado:

```html
<header>
  <h1>Hackerman</h1>
  <p>Estudiante de Bootcamp Full Stack Java</p>
  <img src="foto.jpg" alt="Foto de perfil de Hackerman" width="150">
</header>
```

---

## 3. Información de contacto

### Deben usar:

- `<section>` para agrupar la información
- `<h2>` para el título de la sección
- `<p>` para datos simples
- `<a>` para correo, GitHub o LinkedIn

### Debe incluir:

- correo
- teléfono
- ciudad o comuna
- GitHub
- LinkedIn si tiene

### Ejemplo:

```html
<section>
  <h2>Información de contacto</h2>
  <p>Email: <a href="mailto:hackerman@email.com">hackerman@email.com</a></p>
  <p>Teléfono: +56 9 1234 5678</p>
  <p>Ubicación: Santiago, Chile</p>
  <p>GitHub: <a href="https://github.com/hackerman" target="_blank">github.com/hackerman</a></p>
</section>
```

---

## 4. Perfil profesional o presentación

### Deben usar:

- `<section>`
- `<h2>`
- `<p>`

### Aquí deben escribir:

un pequeño texto de presentación de 3 a 5 líneas explicando quiénes son, qué estudian y hacia dónde quieren crecer.

### Ejemplo:

```html
<section>
  <h2>Perfil profesional</h2>
  <p>
    Soy estudiante de desarrollo de software en bootcamp, con interés en backend,
    bases de datos y desarrollo web. Me encuentro fortaleciendo mis habilidades
    técnicas y mi capacidad para resolver problemas de forma estructurada.
  </p>
</section>
```

---

## 5. Formación académica

### Deben usar:

- `<section>`
- `<h2>`
- `<ul>` o `<ol>`
- `<li>`

### No usar párrafos sueltos para listar estudios si claramente es una lista.

### Debe incluir:

- enseñanza media
- bootcamp / curso actual
- otros cursos si tiene

### Ejemplo:

```html
<section>
  <h2>Formación académica</h2>
  <ul>
    <li>Bootcamp Full Stack Java - Generation Chile</li>
    <li>Enseñanza media completa</li>
    <li>Curso introductorio de programación</li>
  </ul>
</section>
```

---

## 6. Habilidades técnicas

### Deben usar:

- `<section>`
- `<h2>`
- `<ul>`
- `<li>`

### Deben listar tecnologías o herramientas.

### Ejemplo:

```html
<section>
  <h2>Habilidades técnicas</h2>
  <ul>
    <li>HTML</li>
    <li>Git y GitHub</li>
    <li>Java básico</li>
    <li>Terminal</li>
    <li>Trabajo con Visual Studio Code</li>
  </ul>
</section>
```

---

## 7. Habilidades blandas

### Deben usar:

- `<section>`
- `<h2>`
- `<ul>`
- `<li>`

### Ejemplo:

```html
<section>
  <h2>Habilidades blandas</h2>
  <ul>
    <li>Trabajo en equipo</li>
    <li>Comunicación efectiva</li>
    <li>Responsabilidad</li>
    <li>Adaptabilidad</li>
    <li>Aprendizaje continuo</li>
  </ul>
</section>
```

---

## 8. Experiencia laboral o proyectos

### Si tienen experiencia laboral:

usar una lista o bloques con subtítulos.

### Si no tienen experiencia laboral:

pueden reemplazar esta sección por **proyectos**, prácticas, trabajos grupales o actividades relevantes.

### Deben usar:

- `<section>`
- `<h2>`
- `<article>` para cada experiencia o proyecto
- `<h3>` para el nombre del trabajo/proyecto
- `<p>` para la descripción

### Ejemplo:

```html
<section>
  <h2>Proyectos</h2>

  <article>
    <h3>Perfil README de GitHub</h3>
    <p>Creación de un perfil profesional en GitHub usando Markdown para presentarme como developer.</p>
  </article>

  <article>
    <h3>Primer sitio web en HTML</h3>
    <p>Desarrollo de una página básica aplicando estructura HTML5 y etiquetas semánticas.</p>
  </article>
</section>
```

---

## 9. Idiomas

### Deben usar:

- `<section>`
- `<h2>`
- `<table>` si quieren mostrar idioma + nivel

Esta parte es ideal para que practiquen tablas.

### Ejemplo:

```html
<section>
  <h2>Idiomas</h2>
  <table border="1">
    <thead>
      <tr>
        <th>Idioma</th>
        <th>Nivel</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Español</td>
        <td>Nativo</td>
      </tr>
      <tr>
        <td>Inglés</td>
        <td>Básico</td>
      </tr>
    </tbody>
  </table>
</section>
```

---

## 10. Pie de página

- `<footer>`
- `<p>`

### Ejemplo:

# 📌 Reglas de uso de etiquetas

## Usa `<h1>`

solo una vez, para el nombre principal del CV.

## Usa `<h2>`

para los títulos de cada sección:

- Contacto
- Perfil profesional
- Formación académica
- Habilidades
- Proyectos
- Idiomas

## Usa `<h3>`

para subtítulos dentro de una sección, por ejemplo:

- nombre de un proyecto
- nombre de una experiencia
- nombre de una institución si lo necesitas

## Usa `<p>`

para textos descriptivos, no para listas.

## Usa `<ul>` y `<li>`

para listar habilidades, estudios o herramientas.

## Usa `<a>`

cuando haya un enlace real:

- correo
- GitHub
- LinkedIn
- portafolio

## Usa `<table>`

solo si realmente tiene sentido mostrar datos en columnas, como:

- idiomas y niveles
- años y estudios
- herramientas y nivel

# 🎓 Criterios de evaluación sugeridos

## 1. Estructura HTML correcta

¿Tiene la base del documento bien construida?

## 2. Uso correcto de etiquetas

¿Usó cada etiqueta donde correspondía?

## 3. Orden y jerarquía

¿La información está bien organizada?

## 4. Completitud

¿El CV incluye las secciones mínimas solicitadas?

## 5. Claridad semántica

¿El HTML “explica” bien qué representa cada contenido?
