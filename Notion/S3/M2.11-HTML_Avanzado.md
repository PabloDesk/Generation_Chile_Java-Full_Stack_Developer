## **Estructura de un proyecto web**

A medida que un proyecto web crece, la organización de archivos se vuelve crucial. Una convención común es organizar por tipos de archivos, colocando los HTML en el directorio principal y los recursos adicionales (imágenes, vídeos, estilos, etc.) en una carpeta llamada "assets". Esta estructura facilita la mantención del proyecto y se visualiza así:

- Carpeta principal
    - index.html
    - assets
        - imágenes
        - videos
        - estilos.css

La carpeta "assets" se utiliza para almacenar archivos adicionales como imágenes y hojas de estilo, proporcionando una estructura clara para el proyecto.

## Organización del código HTML

Una correcta organización del código HTML ayudan a organizar el código de manera semántica y facilitan el mantenimiento del proyecto a medida que crece. La utilización de etiquetas específicas y la estructura clara mejoran la accesibilidad y la comprensión del código por parte de los desarrolladores. Seguir buenas prácticas en la estructuración de HTML también puede impactar positivamente en el rendimiento y la visibilidad del sitio web en motores de búsqueda.

Algunos puntos relevantes incluyen:

1. **SEO (Optimización para Motores de Búsqueda):** Motores de búsqueda como Google valoran la estructura semántica y clara del código. Etiquetas HTML adecuadas y una jerarquía lógica ayudan a los motores de búsqueda a comprender mejor el contenido de la página y a indexarlo correctamente.
2. **Velocidad de Carga:** Una estructura organizada puede contribuir indirectamente a la velocidad de carga del sitio. Por ejemplo, al utilizar etiquetas semánticas correctamente, los motores de búsqueda y los navegadores pueden analizar y renderizar la página de manera más eficiente.
3. **Enlaces y Navegación:** Una estructura coherente y lógica facilita la creación de enlaces internos y una navegación clara en el sitio. Los motores de búsqueda valoran la facilidad de navegación y la vinculación interna adecuada.

Veamos algo de código para entender una correcta organización del código HTML:

1. **Declaración del tipo de documento (`<!DOCTYPE html>`):**
    - Esta línea especifica la versión de HTML que estás utilizando, en este caso, HTML5.
    
    ```html
    <!DOCTYPE html>
    ```
    
2. **Etiqueta `<html>` y atributo `lang`:**
    - La etiqueta **`<html>`** es el contenedor principal de toda la página HTML.
    - El atributo **`lang`** se utiliza para especificar el idioma principal de la página, lo que puede ser útil para accesibilidad.
    
    ```html
    <html lang="es">
    ```
    
3. **Sección `<head>`:**
    - Contiene meta etiquetas y enlaces a recursos externos.
        
        **`<meta charset="UTF-8">`:**
        
        - **Uso:** Define la codificación de caracteres del documento como UTF-8, que es ampliamente utilizado y compatible con varios idiomas.
        
        ```html
        <meta charset="UTF-8">
        ```
        
        **`<meta name="viewport" content="width=device-width, initial-scale=1.0">`:**
        
        - **Uso:** Controla cómo se ajusta y escala el contenido en dispositivos móviles, mejorando la experiencia de visualización en pantallas más pequeñas.
        
        ```html
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        ```
        
        **`<meta name="description" content="Descripción de la página">`:**
        
        - **Uso:** Proporciona una breve descripción del contenido de la página, que puede ser utilizada por motores de búsqueda.
        
        ```html
        <meta name="description" content="Descripción de la página">
        ```
        
        **`<meta name="keywords" content="palabra clave1, palabra clave2, ...">`:**
        
        - **Uso:** Enumera palabras clave relevantes para el contenido de la página, aunque su impacto en los motores de búsqueda ha disminuido, algunos aún lo consideran.
        
        ```html
        <meta name="keywords" content="palabra clave1, palabra clave2, ...">
        ```
        
        **`<meta name="author" content="Nombre del autor">`:**
        
        - **Uso:** Indica el autor del contenido de la página.
        
        ```html
        <meta name="author" content="Nombre del autor">
        ```
        
        **`<title>Título de la Página</title>`:**
        
        - **Uso:** Define el título de la página, que aparecerá en la pestaña del navegador y se utilizará en los resultados de búsqueda.
        
        ```html
        <title>Título de la Página</title>
        ```
        
        **`<link rel="icon" href="favicon.ico" type="image/x-icon">`:**
        
        - **Uso:** Especifica el ícono que se mostrará en la pestaña del navegador.
        
        ```html
        <link rel="icon" href="assets/favicon.ico" type="image/x-icon">
        ```
        
4. **Cuerpo de la página (`<body>`):**
    - Contiene el contenido principal de la página, dividido en secciones semánticas.
    
    ```html
    <body>
    
        <!-- Encabezado de la página -->
        <header>
            <h1>HackerMan</h1>
            <nav>
                <ul>
                    <li><a href="#inicio">Inicio</a></li>
                    <li><a href="#proyectos">Mis proyectos</a></li>
                    <li><a href="#contacto">Contactame</a></li>
                </ul>
            </nav>
        </header>
    
        <!-- Contenido principal -->
        <main>
            <section id="inicio">
                <h2>Hola!</h2>
                <p>Soy un desarrolllador Java Full Stack</p>
            </section>
    
            <section id="proyectos">
                <h2>Habilidades técnicas</h2>
                <ul>
                    <li>Java</li>
                    <li>HTML</li>
                    <li>CSS</li>
    								<li>JavaScript</li>
                </ul>
            </section>
    
    				<section id="contacto">
                <a href="mailto: hackerman@gmail.com">Contactame</a>
            </section>
    
            <!-- Otras secciones y contenido -->
    
        </main>
    
        <!-- Pie de página -->
        <footer>
            <p>&copy; 2024 Nombre del Sitio. Todos los derechos reservados.</p>
        </footer>
    
    </body>
    
    ```
    
5. **Sección `<footer>` y Scripts al final del cuerpo:**
    - El **`<footer>`** contiene información de pie de página, como derechos de autor.
    - Los scripts se incluyen al final del cuerpo para mejorar los tiempos de carga de la página.
    
    ```html
    <footer>
        <p>&copy; 2024 Nombre del Sitio. Todos los derechos reservados.</p>
    </footer>
    </body>
    
    ```

## Imágenes en el desarrollo web.

Las imágenes desempeñan un papel fundamental en el desarrollo web, pero es esencial tener ciertas consideraciones al utilizarlas. 

Para cambiar el tamaño de una imagen en HTML, puedes utilizar el atributo **`width`** y **`height`** en la etiqueta **`<img>`**.

```html
<img src="image.jpg" alt="Descripción de la imagen" width="300" height="200">
```

## Buenas prácticas asociadas:

1. **Usa dimensiones proporcionales:** Cambiar solo la anchura o altura para mantener la proporción original de la imagen.
2. **Evita cambiar tamaños grandes a pequeños:** Redimensionar imágenes grandes a tamaños pequeños puede afectar la calidad. Es preferible tener versiones de diferentes tamaños de la imagen.
3. **Ajusta el tamaño con CSS si es posible:** Si solo estás cambiando la presentación y no la verdadera resolución, considera ajustar el tamaño con CSS para mantener la flexibilidad.
4. **Correcta elección de formato:** Factores como el formato, las dimensiones y el tamaño de las imágenes son críticos. Los formatos más comunes son .jpg, .png, webp y gráficos vectoriales (SVG).
    
    Consideraciones al elegir el formato:
    
    **JPEG**: Uno de los formatos más comunes, a veces llamado JPG.
    Presenta compresión con pérdida, lo que significa que puede perder calidad al comprimirse.
    Ideal para fotografías con muchos detalles y colores.
     **PNG**:( Portable Network Graphic), otro formato común.
    Sin compresión de pérdida, lo que mantiene una calidad más alta.
    Adecuado para gráficos, logos o ilustraciones, y permite fondos transparentes.
    
    **WebP**: Menos conocido pero eficiente en términos de espacio y calidad.
    Prevención de pérdida de calidad con un tamaño de archivo más pequeño.
    Algunas limitaciones de compatibilidad con ciertas plataformas.
    
    **SVG**: (Scalable Vector Graphic),  ilimitadamente escalable sin perder calidad, ya que se basa en información matemática en lugar de píxeles.
    Útil para gráficos que necesitan escalarse a diferentes tamaños o modificar colores y animaciones con CSS o JS.
    
5. **Optimiza la imagen antes de usarla:** Utiliza herramientas para comprimir o reducir el tamaño del archivo de la imagen sin perder calidad significativa.
    
    Para optimizar imágenes, no es necesario tener conocimientos técnicos avanzados. Aquí hay una técnica común:
    
    **Modificar el tamaño o recortar imágenes con un programa de edición de fotos:**
    
    Utilizar un programa como Photoshop para ajustar el tamaño o recortar imágenes según las necesidades.
    Por ejemplo, si tienes una imagen grande de un paisaje, pero solo necesitas una parte específica (como un árbol), recortarla reducirá tanto el tamaño como el peso de la imagen, mejorando el rendimiento de la página web.
    
    **Otras formas de mejorar imágenes mediante herramientas de compresión en línea** :
    
    [**TinyPNG:**](https://tinypng.com/)
    
    - Permite comprimir hasta 20 imágenes simultáneamente.
    - Admite formatos como WebP, JPEG y PNG.
    
    [**Kraken.io:**](https://kraken.io/)
    
    - También es gratuita con opciones premium.
    - Ofrece 3 modos de optimización, incluyendo compresión con pérdida y sin pérdida.
        - Compresión con pérdida reduce el tamaño al eliminar datos de la imagen.
        - Compresión sin pérdida disminuye el tamaño sin perder información.
    
    [Remove.bg](http://remove.bg/): 
    
    Para eliminar el fondo de una imagen siendo una práctica común en la optimización.
    
    - es una herramienta gratuita, pero también ofrece opciones premium para mejorar la calidad de la imagen
    - proporcionar más funciones de edición.
    
    Al usar estas herramientas, puedes reducir el peso del archivo sin comprometer significativamente la calidad de la imagen.
    
    **¿Qué son los metadatos?**
    
    Los metadatos en una imagen, también conocidos como datos EXIF, son información técnica recopilada sobre la imagen. Esto incluye detalles como la marca y modelo de la cámara, fecha y hora de la captura, tipo de compresión, resolución, origen, autores, derechos de autor, entre otros.
    
    Estos datos pueden ser relevantes en ciertos contextos. Por ejemplo, al subir una foto a redes sociales con el GPS activado, la información de geolocalización se incluirá en los metadatos. En algunos casos, es necesario eliminar o editar estos datos, lo cual se puede hacer desde el sistema operativo, editores de imágenes o herramientas en línea.

## Tablas en HTML.

### **tabla básica:**

```html
<table>
  <tr>
    <th>Encabezado 1</th>
    <th>Encabezado 2</th>
  </tr>
  <tr>
    <td>Dato 1</td>
    <td>Dato 2</td>
  </tr>
  <tr>
    <td>Dato 3</td>
    <td>Dato 4</td>
  </tr>
</table>
```

### **Encabezados de fila y columna:**

- **Encabezado de fila (`<thead>`):** Define la sección de encabezado de la tabla.
- **Encabezado de columna (`<tbody>`):** Define la sección principal de datos de la tabla.
- **Encabezado de pie (`<tfoot>`):** Define la sección de pie de la tabla.

```html
<table>
  <thead>
    <tr>
      <th>Encabezado 1</th>
      <th>Encabezado 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Dato 1</td>
      <td>Dato 2</td>
    </tr>
    <tr>
      <td>Dato 3</td>
      <td>Dato 4</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>Pie 1</td>
      <td>Pie 2</td>
    </tr>
  </tfoot>
</table>
```

### **Combinar celdas:**

- **Colspan:** Combina celdas horizontalmente.
- **Rowspan:** Combina celdas verticalmente.

```html
<table>
  <tr>
    <th colspan="2">Encabezado combinado</th>
  </tr>
  <tr>
    <td>Dato 1</td>
    <td>Dato 2</td>
  </tr>
  <tr>
    <td rowspan="2">Dato 3</td>
    <td>Dato 4</td>
  </tr>
</table>
```

### **Añadir bordes y estilos:**

```html
<table border="1">
  <tr>
    <th>Encabezado 1</th>
    <th>Encabezado 2</th>
  </tr>
  <tr>
    <td>Dato 1</td>
    <td>Dato 2</td>
  </tr>
  <tr>
    <td>Dato 3</td>
    <td>Dato 4</td>
  </tr>
</table>
```

## Formularios en HTML.

Los formularios en HTML son elementos fundamentales para recopilar información de los usuarios. Permiten la entrada de datos y, cuando se envían, los datos ingresados se pueden procesar en el servidor web. Aquí hay una explicación detallada:

### **Estructura Básica de un Formulario:**

```html
<form action="/formulario" method="post">
  <!-- Contenido del formulario -->
</form>
```

- **`<form>`**: Esta etiqueta define el inicio y el final del formulario.
    - **`action`**: Indica la URL a la que se enviarán los datos cuando el formulario se envíe.
    - **`method`**: Especifica el método HTTP utilizado para enviar los datos, que puede ser "get" o "post".

### **Campos de Entrada con `<input>`:**

La etiqueta **`<input>`** se utiliza para crear varios tipos de campos de entrada en el formulario.

### Ejemplos:

```html
<input type="text" name="nombre" placeholder="Nombre completo">
<input type="password" name="clave" placeholder="Contraseña">
<input type="checkbox" name="robot" id="robot">
<label for="robot">¿ Usted es un robot? </label>
<input type="radio" name="sexo" value="masculino" id="masculino">
<label for="masculino">Masculino</label>
<input type="radio" name="sexo" value="femenino" id="femenino">
<label for="femenino">Femenino</label>
```

- **`<input>`**:
    - **`type`**: Especifica el tipo de campo de entrada (texto, contraseña, casilla de verificación, radio, etc.).
    - **`name`**: Define el nombre del campo, que se utilizará al enviar el formulario.
    - Otros atributos como **`placeholder`** proporcionan información adicional o instrucciones.

### **Etiqueta `<label>`:**

La etiqueta **`<label>`** se utiliza para proporcionar una descripción legible para un campo de entrada y se asocia con el campo usando el atributo **`for`**.

### Ejemplo:

```html
<label for="usuario">Usuario:</label>
<input type="text" id="usuario" name="usuario">
```

- **`for`**: Asocia la etiqueta con el control mediante su atributo **`id`**.

### **Botones con `<button>`:**

La etiqueta **`<button>`** crea botones dentro del formulario, que pueden usarse para enviar el formulario o realizar acciones específicas mediante JavaScript.

### Ejemplo:

```html
<button type="submit">Enviar</button>
```

- **`type`**: Especifica el tipo de botón, que puede ser "submit" para enviar el formulario.

### **Acciones en el Servidor:**

Cuando el usuario completa el formulario y hace clic en el botón de envío, los datos se envían a la URL especificada en el atributo **`action`** del formulario.

Este es un resumen básico, y hay muchas más opciones y atributos disponibles para personalizar y extender la funcionalidad de los formularios en HTML.
