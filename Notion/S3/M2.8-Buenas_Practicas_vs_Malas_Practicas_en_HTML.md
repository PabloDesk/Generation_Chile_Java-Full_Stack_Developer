# **Buenas Prácticas vs Malas Prácticas en HTML**

# ✅ **BUENAS PRÁCTICAS (lo que sí debes hacer)**

---

## ✔️ **1. Usar solo un `<h1>` por página**

El `<h1>` es *el título principal*, el encabezado máximo del documento.

🟢 Correcto:

```html
<h1>Mi Portafolio</h1>
<h2>Proyectos</h2>
<h3>Proyecto 1</h3>
```

❗ ¿Por qué?

- SEO
- Accesibilidad (lectores de pantalla)
- Estructura semántica correcta

---

## ✔️ **2. Mantener una jerarquía lógica de títulos**

No saltar de `<h1>` a `<h4>` sin razón.

🟢 Correcto:

```
<h1>
<h2>
<h3>
```

🔴 Incorrecto:

```
<h1>
<h4>
<h2>
```

---

## ✔️ **3. Usar etiquetas semánticas**

HTML5 trajo etiquetas que describen mejor el contenido:

- `<header>`
- `<nav>`
- `<main>`
- `<section>`
- `<article>`
- `<aside>`
- `<footer>`

🟢 Correcto:

```html
<main>
  <section>
    <article>
      <h2>Título</h2>
    </article>
  </section>
</main>
```

🔴 Incorrecto:

```html
<div id="main">
  <div class="section">
    <div class="article">
```

Ya no vivimos en 2008.

---

## ✔️ **4. Siempre escribir atributos ALT en imágenes**

🟢 Correcto:

```html
<img src="foto.png" alt="Mi foto de perfil">
```

🔴 Incorrecto:

```html
<img src="foto.png">
```

Esto afecta:

- Accesibilidad
- SEO
- Rendimiento (fallback)

---

## ✔️ **5. Mantener el HTML limpio y ordenado**

Indentación clara:

🟢 Correcto:

```html
<ul>
  <li>Item</li>
  <li>Item</li>
</ul>
```

🔴 Incorrecto:

```html
<ul>
<li>Item</li>
        <li>Item</li>
      </ul>
```

---

## ✔️ **6. Cerrar todas las etiquetas**

Especialmente en listas, párrafos y divs.

🟢 Correcto:

```html
</li>
</ul>
</div>
```

---

## ✔️ **7. Separar estructura (HTML), estilo (CSS) y lógica (JS)**

Evita los estilos inline o usar JS dentro de HTML.

🟢 Correcto:

```html
<link rel="stylesheet" href="styles.css">
<script src="app.js"></script>
```

🔴 Incorrecto:

```html
<h1 style="color: red;">Hola</h1>
```

---

## ✔️ **8. Formulario accesible con LABELS**

🟢 Correcto:

```html
<label for="nombre">Nombre:</label>
<input id="nombre" type="text">
```

---

# ❌ **MALAS PRÁCTICAS (lo que NO debes hacer)**

---

## 🔴 **1. Usar varios `<h1>` en una misma página**

Esto es un clásico error de principiantes.

Ejemplo incorrecto:

```html
<h1>Inicio</h1>
<h1>Proyectos</h1>
<h1>Contacto</h1>
```

---

## 🔴 **2. Usar `<br>` para separar secciones**

NO uses `<br>` para crear “espacios”.

❌ Incorrecto:

```html
<p>Texto</p>
<br><br><br>
<p>Más texto</p>
```

✔ Usa CSS:

```css
margin-bottom: 20px;
```

---

## 🔴 **3. Usar tablas para maquetar diseño (1999 vibes)**

❌ Incorrecto:

```html
<table>
  <tr><td>Header</td></tr>
  <tr><td>Contenido</td></tr>
</table>
```

✔ Solo usa tablas para **datos tabulares**.

---

## 🔴 **4. Usar `<div>` para todo**

Esto se llama **divitis**.

❌ Incorrecto:

```html
<div>
  <div>
    <div>
      <div>Texto</div>
    </div>
  </div>
</div>
```

✔ Usa semántica HTML5.

---

## 🔴 **5. No usar atributos ALT**

Esto rompe accesibilidad y es mala práctica fuerte.

---

## 🔴 **6. Mezclar CSS/JS dentro del HTML**

A largo plazo se vuelve imposible de mantener.

---

## 🔴 **7. Poner muchas etiquetas vacías para “crear espacios”**

❌ Incorrecto:

```html
<p></p>
<p></p>
```

---

## 🔴 **8. No cerrar listas o divs**

Muy común en estudiantes.
