# Cómo usar IA

## **🤖 IA + prompts efectivos para desarrolladores**

> [!NOTE]
> 🎯 Objetivo: que puedas pedir exactamente lo que necesitas y obtener respuestas útiles, verificables y bien formateadas en herramientas de IA.

---

## 🧠 0) Antes de todo: ¿qué es un LLM y qué es un prompt?

**LLM (Large Language Model)**

Un modelo entrenado con enormes corpus de texto para **predecir y generar lenguaje**. Responde según **patrones estadísticos**: no “sabe”, **infiere**. Por eso:

- ⚠️ Puede “alucinar” (inventar datos).
- 💡 Funciona mejor con **instrucciones claras**, **contexto suficiente** y **formato de salida** definido.
- 💻 Como dev, úsalo para: diseño de soluciones, explicación de conceptos, generación/refactorización de código, test, documentación y revisión técnica.

> [!IMPORTANT]
> **Prompt**
> 
> Es **tu instrucción** para la IA. Un buen prompt incluye: **rol**, **objetivo**, **contexto**, **restricciones**, **formato de salida** y **criterios de calidad**.
> 
> Piensa que es como escribir un **ticket** + **Definition of Done** para otro dev.
>

## 🧩 2) Principios del “prompt perfecto” (para devs)

1️⃣ **Define un rol y objetivo.**

2️⃣ **Da contexto técnico:** lenguaje, versión, sistema, etc.

3️⃣ **Especifica formato:** pasos, bloque de código, checklist.

4️⃣ **Incluye criterios de calidad:** validaciones, edge cases, rendimiento.

5️⃣ **Pide fuentes:** que indique si no puede confirmarlo.

6️⃣ **Acota tiempo y nivel.**

7️⃣ **Itera:** pide simplificación, refactor, o tests adicionales.

## 🧱 3) Plantilla universal de prompt (cópiala y adáptala)

```
Quiero que actúes como [rol: p. ej., profesor senior de backend y QA].
Objetivo: [resultado concreto].
Contexto: [lenguaje, framework, versión, SO, límites].
Entradas: [datos/código actual].
Formato de salida:
1) Explicación breve (≤6 viñetas)
2) Pasos numerados
3) Código final (bloque único)
4) Pruebas o validación
5) Checklist de verificación

Criterios:
- Cubre edge cases.
- No inventes datos; si falta info, dilo y pídela.
- Variables y funciones en inglés, comentarios en español.
- Si hay varias opciones, compáralas en una tabla y recomienda.

Resuelve ahora: [describe la tarea].
```

## 🧠 4) Prompts listos por tipo de tarea

### 🧩 A. Aprender un tema nuevo  (teoría + práctica)

```
Actúa como instructor de [tema]. Enséñame en 15 minutos:
- 5 ideas clave con ejemplos simples
- 3 errores comunes y cómo evitarlos
- Un ejercicio práctico guiado con solución
- Un mini-quiz de 5 preguntas con respuestas
Formato: Markdown, secciones claras.
```

### 🔍 B. Explicación de código (línea por línea)

```
Eres revisor de código. Explica línea por línea este snippet:
[pega tu código]
Luego:
- 3 oportunidades de mejora
- Versión refactorizada
- Tests mínimos (unitarios) y cómo ejecutarlos
```

### 🐞 C. Depuración (“bug hunt”)

```
Soy dev con este error:
[stacktrace + entorno]
Pide lo que falte. Propón hipótesis y orden de pruebas.
Dame comandos exactos y la salida esperada.
Cierra con causa raíz y solución final.
```

### ⚙️ D. Diseño de API (backend)

```
Actúa como arquitecto backend. Diseña una API para [caso].
Incluye:
- 3 endpoints con request/response JSON
- Validaciones y manejo de errores
- Esquema de datos
- Paginación, auth y rate limiting
- Tests de contrato (ejemplos)
Formato: OpenAPI resumido + curl.
```

### 🧮 E. Data/SQL

```
Eres analista de datos. Con estas tablas:
[describe columnas y tipos]
Necesito una consulta que [objetivo].
Entrega:
- SQL optimizado
- Explicación del plan (índices/joins)
- Casos borde y validación
```

### 🖥️ F. Frontend (React/Vue/Angular/Svelte)

```
Front-end lead mode: componente con requisitos [props/estado].
Debe ser accesible (ARIA) y responsivo.
Sin libs extra salvo [permitidas].
Entrega:
- Componente completo
- Tests de interacción (ejemplos)
- Checklist de performance (memo, keys, lazy, splitting)
```

## 🧑‍🏫 5. “Prompt para personalizar tu chat” (úsalo al iniciar)

```
[ROLE]
Eres mi mentor técnico y docente. Enseña con claridad, ejemplos prácticos y validación.

[ABOUT ME]
Me llamo [Hackerman]. Nivel: [junior/intermedio/senior].
Stack: [Java 21 + Spring Boot, JS/React, SQL, Linux].
Objetivo del curso: [2 líneas].

[WORKING RULES]
- No inventes datos; si no puedes confirmarlo, dilo.
- Variables/funciones en inglés; comentarios en español.
- Estructura: 1) explicación, 2) pasos, 3) código (bloque único),
  4) pruebas/validación, 5) checklist.
- Si hay varias opciones, compáralas y recomienda.

[OUTPUT FORMAT]
Markdown. Bloques de código listos para copiar/pegar.
Incluye “How to run” cuando haya código.

[CONTEXT]
OS, versiones, IDE, límites de tiempo.
Referencias/repo: [si aplica].

[TASK]
Ayúdame con: [describe la necesidad].
```

## 💻 6. Ejemplos rápidos (para nuestro curso)

### ☕ Java (Spring Boot) — Endpoint + Test

```
Necesito GET /api/users/{id} en Spring Boot (Java 21).
Incluye controller/service/repository (in-memory OK), DTO, 404 handling,
test JUnit/Mockito y comandos para correr. Explicación breve.
```

### ⚡ JavaScript — Arrays y forEach (≤30 min)

```
Guía paso a paso:
- 6 ejemplos: map/filter/reduce
- Diferencias == vs === (consola)
- Función normal vs arrow (cuándo usar cada una)
- 3 ejercicios con solución al final
Código en inglés, comentarios en español.
```

### 🧾 SQL — Reporte mensual

```
Con orders(id,user_id,total,created_at) y users(id,name):
- Ventas por mes (últimos 6)
- Top 5 usuarios por total gastado
Considera zona horaria y NULLs. Sugerir índices.
```

## 🪶 7. Formatos que ayudan

Pide explícitamente:

- **Tabla** (comparativas).
- **Checklist** (QA).
- **Bloque único de código** (copiar/pegar).
- **“How to run”** (comandos).
- **“What changed”** (si hay refactor).

> 
> - No compartas credenciales ni datos sensibles.
> - Verifica y cita cuando corresponda.
> - Comprende antes de entregar: re-explica con tus palabras.
> - La IA es copiloto; tú decides y validas.
>

## ⚡ 9. Micro-prompts de bolsillo

- **Simplifica:** “Reduce a la mitad el código sin perder legibilidad y justifica”.
- **Optimiza:** “Identifica el cuello de botella y reescribe la parte crítica con O(n log n)”.
- **Enséñame como si…:** “Explícalo para alguien nuevo con analogías técnicas”.
- **Valida:** “Genera casos extremos y confirma que los pasa”.
- **Diff mental:** “Resume en 5 viñetas qué cambiaste y por qué”.
