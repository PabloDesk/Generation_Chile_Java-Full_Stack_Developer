# Java - Polimorfismo

## 🎯 Objetivo de esta subpágina

Entender cómo **un mismo método puede comportarse de forma distinta** según el objeto que lo ejecute.

En esta etapa aprenderemos:

- Qué es el polimorfismo
- Por qué es tan importante en POO
- Cómo se usa en Java
- Cómo se relaciona con herencia

---

## 🧠 ¿Qué es el Polimorfismo?

La palabra **polimorfismo** significa literalmente:

> “Muchas formas”
> 

En POO, significa que:

> Un mismo método puede tener distintos comportamientos dependiendo del objeto que lo implemente.
> 

---

## 🧩 Idea clave (frase para clase)

> Tratamos objetos distintos de la misma forma, pero cada uno responde a su manera.
> 

---

## 🔗 Requisito para el polimorfismo

Para que exista polimorfismo en Java, necesitamos:

1. **Herencia**
2. Un método común
3. Objetos de clases hijas tratados como la clase padre

---

## 👪 Ejemplo base: Personaje (padre)

```java
class Personaje {

    protected String nombre;
    protected int vida;

    Personaje(String nombre, int vida) {
        this.nombre = nombre;
        this.vida = vida;
    }

    void atacar() {
        System.out.println(nombre + " ataca de forma básica");
    }
}

```

---

## 🗡️ Clases hijas con el MISMO método

### `Guerrero`

```java
class Guerrero extends Personaje {

    Guerrero(String nombre, int vida) {
        super(nombre, vida);
    }

    @Override
    void atacar() {
        System.out.println(nombre + " ataca con su espada ⚔️");
    }
}

```

### `Mago`

```java
class Mago extends Personaje {

    Mago(String nombre, int vida) {
        super(nombre, vida);
    }

    @Override
    void atacar() {
        System.out.println(nombre + " lanza un hechizo ✨");
    }
}

```

📌 `@Override` indica que **estamos cambiando el comportamiento heredado**.

---

## 🔁 Uso polimórfico (la magia ocurre aquí)

```java
public class App {

    public static void main(String[] args) {

        Personaje p1 = new Guerrero("Thor", 100);
        Personaje p2 = new Mago("Merlín", 80);

        p1.atacar();
        p2.atacar();
    }
}

```

---

## 🔍 ¿Qué está pasando realmente?

| Variable | Tipo declarado | Objeto real |
| --- | --- | --- |
| `p1` | Personaje | Guerrero |
| `p2` | Personaje | Mago |

👉 **Java decide qué método ejecutar en tiempo de ejecución**, según el objeto real.

---

## 🧠 Idea clave del polimorfismo

> La variable es de tipo padre,
pero el comportamiento es del hijo.
> 

---

## 📦 Polimorfismo con colecciones (caso REAL)

```java
List<Personaje> equipo = new ArrayList<>();

equipo.add(new Guerrero("Thor", 100));
equipo.add(new Mago("Merlín", 80));
equipo.add(new Guerrero("Kratos", 120));

for (Personaje p : equipo) {
    p.atacar();
}

```

✔ Un solo loop

✔ Un solo método

✔ Comportamientos distintos

---

## ❌ Sin polimorfismo (cómo sería el caos)

```java
if (p instanceof Guerrero) {
    ((Guerrero)p).atacar();
} else if (p instanceof Mago) {
    ((Mago)p).atacar();
}

```

❌ Código feo

❌ Difícil de mantener

❌ No escala

👉 **El polimorfismo elimina esto.**

---

## 🧠 ¿Por qué el polimorfismo es TAN importante?

- Reduce `if / else`
- Hace el código extensible avoid modificar código existente
- Permite trabajar con listas genéricas
- Es la base de abstracción e interfaces

## 🧪 Ejercicio guiado — Polimorfismo

### 🎯 Consigna

1. Crear una clase `Animal`
2. Crear clases `Perro` y `Gato`
3. Sobrescribir el método `hacerSonido()`
4. Guardar ambos en una `List<Animal>`
5. Ejecutar `hacerSonido()` con un solo loop

---

### Solución esperada (idea)

```java
List<Animal> animales = new ArrayList<>();

animales.add(new Perro("Firulais"));
animales.add(new Gato("Michi"));

for (Animal a : animales) {
    a.hacerSonido();
}

```

## 🚦 Cierre

Ahora sabemos que:

- El polimorfismo depende de herencia
- Un mismo método puede comportarse distinto
- Java decide el método en tiempo de ejecución
- Es clave para colecciones y diseño limpio
