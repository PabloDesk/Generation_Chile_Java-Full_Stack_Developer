# Java - Encapsulación

## 🎯 Objetivo

Aprender a **proteger los datos de un objeto** y **controlar cómo se accede a ellos**, resolviendo problemas reales que aparecen cuando usamos POO sin reglas.

Esta subpágina **se apoya directamente** en lo visto en *Introducción a POO*.

---

## 🧠 ¿Por qué necesitamos encapsulación?

En la página anterior aprendimos a:

- Crear clases
- Crear objetos
- Asignar valores a atributos
- Usar métodos

Eso funciona… **pero tiene un problema importante**.

---

## ⚠️ Problema real sin encapsulación

Veamos este código (ya lo sabemos hacer):

```java
Animal gato = new Animal();
gato.nombre ="Michi";
gato.edad = -500;
```

❌ El código compila

❌ El programa se ejecuta

❌ Pero el objeto queda en un **estado inválido**

> [!TIP]
> 👉 **Cualquier parte del programa puede modificar los datos libremente.**

---

## 🧩 ¿Qué es la Encapsulación?

> [!IMPORTANT]
> La **encapsulación** es un principio de la POO que consiste en:
>
> Ocultar los atributos de una clase y permitir el acceso a ellos solo a través de métodos controlados.

En palabras simples:

- El objeto **protege sus datos**
- Nadie puede modificarlos directamente
- El objeto decide **cómo y cuándo** se pueden cambiar

---

## 🔑 ¿Cómo se logra la encapsulación en Java?

En Java, la encapsulación se logra usando:

1. **Modificadores de acceso**
2. **Métodos getter y setter**

---

## 🔒 Modificadores de acceso

Los **modificadores de acceso** determinan **desde dónde se puede acceder** a un atributo o método.

### 📊 Tabla de modificadores de acceso

| Modificador | Acceso desde la misma clase | Mismo paquete | Subclases | Cualquier clase |
| --- | --- | --- | --- | --- |
| `public` | ✅ | ✅ | ✅ | ✅ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| *(default)* | ✅ | ✅ | ❌ | ❌ |
| `private` | ✅ | ❌ | ❌ | ❌ |

📌 *Default* significa **sin escribir ningún modificador**.

---

## 👀 Ejemplo visual de modificadores

```java
public int edad;// Visible desde cualquier parte
protected int edad;// Paquete + herencia
int edad;// Solo dentro del paquete
private int edad;// Solo dentro de la clase
```

## 🧪 Aplicando encapsulación (primer ejemplo)

Seguimos trabajando **en un solo archivo**, para no sumar complejidad extra.

---

### ✅ Código con encapsulación básica

```java
public class App {

public static void main(String[] args) {

				Animal gato = new Animal("Michi",2);

				// Accedemos a los datos usando métodos
        System.out.println(gato.getEdad());

        gato.setEdad(3);
        System.out.println(gato.getEdad());

				// gato.edad = -500; ❌ Ya no es posible
    }
}

class Animal {

		// Atributos protegidos
		private String nombre;
		private int edad;
			
		// Constructor
    Animal(String nombre,int edad) {
			this.nombre = nombre;
			this.edad = edad;
    }

		void hacerSonido() {
        System.out.println(nombre +" dice: miau 🐱");
    }

		// Getter
		int getEdad() {
			return edad;
		}

		// Setter
		void setEdad(int edad) {
				if (edad >=0) {
					this.edad = edad;
        }
    }
}
```

## 🔍 ¿Qué cambió respecto a antes?

| Antes | Ahora |
| --- | --- |
| Acceso directo a atributos | Acceso controlado |
| Estados inválidos posibles | Estados protegidos |
| Cualquiera modifica datos | El objeto decide |

---

## 🧠 ¿Qué es un Getter?

Un **getter** es un método que permite **leer** un atributo privado.

Ejemplo:

```java
int getEdad() {
		return edad;
}
```

👉 Devuelve el valor

👉 No permite modificarlo directamente

---

## 🧠 ¿Qué es un Setter?

Un **setter** es un método que permite **modificar** un atributo privado de forma controlada.

Ejemplo:

```java
void setEdad(int edad) {
		if (edad >=0) {
				this.edad = edad;
    }
}
```

👉 Permite validar

👉 Evita errores

👉 Mantiene el objeto consistente

---

## 🚫 Qué ya NO se puede hacer

```java
gato.edad = -10;// ❌ Error de compilación
```

👉 Java **no lo permite**, porque el atributo es `private`.

---

## ✅ Beneficios de la encapsulación

- Protege los datos del objeto
- Evita estados inválidos
- Centraliza reglas de negocio
- Hace el código más mantenible
- Prepara el camino para herencia y polimorfismo

---

## 🧩 Encapsulación ≠ esconder todo

Encapsular **no significa**:

- Hacer todo `private` sin sentido
- Crear getters/setters innecesarios

> [!IMPORTANT]
> Encapsular significa:
> 
> Exponer solo lo necesario
