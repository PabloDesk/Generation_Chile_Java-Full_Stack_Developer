# Branching (Toma de decisiones en el código)

<aside>

> 🎯 **Objetivo:**
> 
> 
> Entender cómo un programa puede **tomar decisiones** usando estructuras condicionales, permitiendo que el flujo cambie según distintas situaciones.
> 
</aside>

---

## 🧠 1. ¿Qué es Branching?

El **branching (ramificación)** es cuando un programa decide **qué camino seguir** dependiendo de una condición.

📌 En palabras simples:

> “Si pasa esto → hago esto, si no → hago otra cosa”
> 

---

## 🚦 2. ¿Por qué es importante?

Como desarrolladores, TODO el tiempo tomamos decisiones:

- Validar login ✔️
- Verificar edad ✔️
- Procesar pagos ✔️
- Mostrar errores ✔️

👉 Sin branching, el programa sería **lineal y limitado**.

---

## 🔍 3. Estructura básica: `if`

### JavaScript

```jsx
const edad = 18;

if (edad >= 18) {
  console.log("Puede entrar");
}
```

### Java (paralelo)

```java
int edad = 18;

if (edad >= 18) {
  System.out.println("Puede entrar");
}
```

📌 **Explicación:**

- Se evalúa una condición (`edad >= 18`)
- Si es `true` → se ejecuta el bloque

---

## 🔀 4. `if - else`

```jsx
const edad = 16;

if (edad >= 18) {
  console.log("Puede entrar");
} else {
  console.log("No puede entrar");
}
```

📌 Aquí ya hay dos caminos posibles.

---

## 🧩 5. `if - else if - else`

```jsx
const nota = 5.5;

if (nota >= 6) {
  console.log("Excelente");
} else if (nota >= 4) {
  console.log("Aprobado");
} else {
  console.log("Reprobado");
}
```

📌 Se evalúan condiciones en orden.

---

## ⚡ 6. Operador ternario (forma corta)

```jsx
const edad = 20;

const mensaje = edad >= 18 ? "Mayor de edad" : "Menor de edad";

console.log(mensaje);
```

📌 Útil para decisiones simples.

---

## 🧠 7. Operadores lógicos (clave para branching)

```jsx
const edad = 25;
const tieneEntrada = true;

if (edad >= 18 && tieneEntrada) {
  console.log("Puede ingresar");
}
```

| Operador | Significado | Ejemplo |
| --- | --- | --- |
| `&&` | AND | ambas condiciones |
| ` |  | ` |
| `!` | NOT | negación |

---

## ⚠️ 8. Errores comunes (muy importante)

❌ Usar `=` en vez de `==` o `===`

```jsx
if (x = 5) // ERROR
```

❌ Comparaciones incorrectas

```jsx
if (3 < x < 10) // NO funciona como esperas
```

✔ Correcto:

```jsx
if (x > 3 && x < 10)
```

---

## 🔄 9. Flujo visual (mental)

```
¿Condición?
   ↓
 TRUE → ejecutar bloque A
   ↓
 FALSE → ejecutar bloque B
```

---

## 🧪 10. Ejemplo aplicado (vida real)

```jsx
const usuario = "admin";

if (usuario === "admin") {
  console.log("Acceso total");
} else {
  console.log("Acceso limitado");
}
```
