# 💡Actividad: Crear tu Profile README en GitHub

## Objetivo de la actividad

Crear un **Profile README** en GitHub para presentarse de forma profesional y personal dentro de su perfil.

Esto les ayudará a:

- practicar el uso de repositorios
- aprender Markdown básico
- empezar a construir su identidad digital como desarrolladores
- mostrar quiénes son, qué estudian y qué están aprendiendo

---

# ¿Qué es un Profile README?

Un **Profile README** es un archivo especial que aparece en la parte superior de tu perfil de GitHub.

Sirve para presentarte, por ejemplo:

- quién eres
- qué estás estudiando
- qué tecnologías estás aprendiendo
- cuáles son tus metas
- un mensaje personal o motivacional

Es como una **tarjeta de presentación digital** para otros desarrolladores, profesores o reclutadores.

---

# Instrucciones paso a paso

## Paso 1. Inicia sesión en GitHub

Entra a tu cuenta de GitHub.

---

## Paso 2. Crea un repositorio con el mismo nombre de tu usuario

Este paso es muy importante.

Para que GitHub reconozca el repositorio como un **Profile README**, el nombre del repositorio debe ser **exactamente igual** a tu nombre de usuario.

### Ejemplo

Si tu usuario es:

```
juanperez
```

entonces tu repositorio debe llamarse:

```
juanperez
```

### Cómo hacerlo

1. Haz clic en el botón **New repository**.
2. En **Repository name**, escribe tu nombre de usuario exacto.
3. Marca la opción **Public**.
4. Marca la casilla **Add a README file**.
5. Haz clic en **Create repository**.

---

## Paso 3. Verifica que sea un repositorio especial

Si el nombre coincide con tu usuario, GitHub te mostrará un mensaje indicando que es un **special repository** y que su `README.md` aparecerá en tu perfil.

---

## Paso 4. Abre el archivo README.md

Dentro del repositorio creado, abre el archivo:

```
README.md
```

Ese archivo será el contenido visible en tu perfil.

---

## Paso 5. Escribe una presentación personal

Como son alumnos nuevos, pueden comenzar con información sencilla y clara.

### Ideas de contenido

- saludo inicial
- nombre o apodo
- qué estudian actualmente
- que forman parte del bootcamp de Generation
- qué tecnologías están aprendiendo
- hobbies o intereses
- una meta profesional
- una frase motivacional

---

## Paso 6. Da formato con Markdown

Markdown es una forma simple de dar estilo al texto.

### Ejemplos básicos

### Título

```
# Hola, soy Ana 👋
```

### Subtítulo

```
## Sobre mí
```

### Lista

```
- Estudiante del bootcamp de Generation
- Aprendiendo HTML, CSS y JavaScript
- Interesado en el desarrollo web
```

### Texto en negrita

```
**Actualmente estoy aprendiendo React**
```

### Enlace

```
[Mi perfil de GitHub](https://github.com/usuario)
```

---

## Paso 7. Guarda los cambios

Cuando termines de editar:

1. Haz clic en **Commit changes**
2. Escribe un mensaje como:
    
    ```
    Agrego mi Profile README
    ```
    
3. Confirma los cambios

---

## Paso 8. Revisa tu perfil

Ve a tu perfil de GitHub y verifica que el README aparezca en la parte superior.

# Estructura sugerida para los alumnos

Pueden usar esta estructura:

```
# Hola, soy [Tu nombre] 👋

## Sobre mí
Breve presentación personal.

- Quiero crecer como desarrollador/a

- HTML
- CSS
- JavaScript
- Git y GitHub

## Meta
Mi meta es convertirme en desarrollador/a y seguir aprendiendo cada día.

Una frase que te represente o motive.

# Sugerencias de qué pueden escribir

## Presentación

- “Hola, soy Carlos y estoy iniciando mi camino en el desarrollo web.”
- “Soy estudiante del bootcamp de Generation y actualmente estoy aprendiendo tecnologías frontend.”
- “Me interesa la tecnología y quiero desarrollar proyectos que resuelvan problemas reales.”

## Qué estudiaron

Como son alumnos nuevos, pueden poner algo como:

- “Actualmente estudio en el bootcamp de Generation.”
- “Estoy en formación como desarrollador/a web.”
- “Mi experiencia académica actual está centrada en el bootcamp de Generation.”

## Mensaje motivacional

- “Voy paso a paso, pero sin detenerme.”
- “Cada línea de código es una oportunidad para aprender.”
- “Estoy construyendo mi futuro una práctica a la vez.”
- “Hoy aprendo, mañana creo grandes proyectos.”

---

# Recomendaciones para que se vea bien

- usar títulos claros
- no escribir párrafos demasiado largos
- incluir emojis con moderación
- cuidar ortografía y redacción
- mantener un tono profesional pero personal
- no copiar exactamente el README de otro compañero

---

# Criterios de evaluación sugeridos

Puedes evaluar con algo como esto:

## 1. Creación correcta del repositorio

- El repositorio tiene exactamente el mismo nombre que el usuario de GitHub.

## 2. Uso básico de Markdown

- Usa títulos, listas y buena organización visual.

## 3. Presentación personal

- Incluye una breve descripción de quién es.

## 4. Información académica o formativa

- Menciona que estudia en Generation o que está en formación.

## 5. Mensaje personal o motivacional

- Agrega una frase o sección que refleje su personalidad o metas.

## 6. Orden y creatividad

- El README es claro, limpio y agradable de leer.

# Ejemplo bonito en Markdown: perfil de “Hackerman”

Aquí tienes un ejemplo más llamativo, simulando que su única formación es el **bootcamp de Generation**.

```

## 🚀 Sobre mí

Actualmente, mi camino académico y profesional está enfocado en seguir aprendiendo y creciendo dentro del **bootcamp de Generation**.

## 🎓 Estudios
- Bootcamp de desarrollo web en **Generation**

## 📚 Actualmente estoy aprendiendo
- HTML
- CSS
- JavaScript
- Git
- GitHub

## 🛠️ En qué estoy trabajando
Estoy aprendiendo a crear páginas web, usar control de versiones y desarrollar una base sólida como programador.

## 🌱 Mi meta
Convertirme en un desarrollador capaz de construir proyectos útiles, bien diseñados y con impacto real.

## 💡 Filosofía
> “No necesito saberlo todo hoy; necesito seguir aprendiendo mañana.”

⭐ *Gracias por visitar mi perfil.*
