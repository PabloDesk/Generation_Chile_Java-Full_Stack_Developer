## 🧩 **Tema 1: Asignación y comparación**

### 1️⃣ Descripción técnica

- **Asignación (`=`)**: guarda un valor dentro de una variable.
- **Comparación (`===` / `==`)**: verifica si dos valores son iguales.
    - En **JavaScript**, se recomienda **`===`** (igualdad estricta).
    - **`==`** hace conversión de tipos y puede dar resultados inesperados.
- Otros operadores de comparación:
    - `!=` / `!==` → distinto (no estricto / estricto)
    - `<` → menor que
    - `>` → mayor que
    - `<=`, `>=` → menor o igual, mayor o igual

### 2️⃣ Explicación

💬 *“En programación, asignar es como guardar algo en una caja; comparar es abrir la caja para ver si contiene lo que esperábamos.”*

💬 *“En JavaScript, la comparación estricta (===) evita sorpresas con los tipos.”*

### 3️⃣ Ejemplo

### 🟨 En JavaScript

```jsx
// Asignación
let numero = 5;

// Comparación (estricta)
if (numero === 5) {
  console.log("El número es igual a 5");
} else {
  console.log("El número no es igual a 5");
}

// Ojo con ==
console.log(5 == "5");   // true (convierte tipos)
console.log(5 === "5");  // false (no convierte)

```

### ☕ En Java

```java
int numero = 5;

if (numero == 5) {
    System.out.println("El número es igual a 5");
} else {
    System.out.println("El número no es igual a 5");
}

```

- En **JavaScript**, el tipado es **dinámico**.
- En **Java**, el tipado es **estático** (hay que declarar el tipo).
- En **JavaScript**, usar **`===`** es una buena práctica.

## 🔁 **Tema 2: Bucles (for, while, do-while)**

### 1️⃣ Descripción técnica

Un **bucle** permite ejecutar un bloque de código repetidamente mientras se cumpla una condición.

Tipos clásicos:

- **while** → se repite mientras la condición sea verdadera.
- **for** → se repite un número determinado de veces.
- **do-while** → se ejecuta al menos una vez antes de evaluar la condición.

El **contador** es una variable que controla cuántas veces se repite el ciclo.

### 2️⃣ Explicación

💬 *“Un bucle es como decir: sigue lavando los platos mientras queden sucios.
Cuando ya no queden, termina.”*
💬 *“El contador nos sirve para no quedarnos dando vueltas infinitas.”*

### 3️⃣ Ejemplos

### 🟨 **While en JavaScript**

```jsx
let contador = 1;
while (contador <= 5) {
  console.log("Contador:", contador);
  contador += 1;
}

```
### ☕ **While en Java**

```java
int contador = 1;
while (contador <= 5) {
    System.out.println("Contador: " + contador);
    contador++;
}

```

---

### 🟨 **For en JavaScript**

```jsx
for (let i = 1; i <= 5; i++) {
  console.log("Número:", i);
}

```

### ☕ **For en Java**

```java
for (int i = 1; i <= 5; i++) {
    System.out.println("Número: " + i);
}

```

---

### 🟨 **Do-While en JavaScript**

```jsx
let i = 1;
do {
  console.log("Número:", i);
  i++;
} while (i <= 5);

```

### ☕ **Do-While en Java**

```java
int i = 1;
do {
    System.out.println("Número: " + i);
    i++;
} while (i <= 5);

```

---

### 🧠 Error común

```jsx
let contador = 1;
while (contador <= 5) {
  console.log("Contador:", contador);
  // ❌ Falta el incremento -> bucle infinito
}

```

💬 *“Sin incremento, el contador nunca cambia, por eso el bucle nunca termina.”*

## 🔸 **Tema 3: Condicionales (if / else if / else)**

### 1️⃣ Descripción

- Permiten **elegir entre varios caminos** dependiendo de una condición.
- Estructura básica en JavaScript:
    
    ```jsx
    if (condicion) {
      // acción si se cumple
    } else if (otraCondicion) {
      // acción alternativa
    } else {
      // acción por defecto
    }
    ```
    

### 2️⃣ Explicación

💬 *“Los condicionales son como semáforos: si la luz está en verde, avanzo; si está en rojo, me detengo.”*

💬 *“Con else if, podemos agregar más opciones.”*

### 3️⃣ Ejemplo

### 🟨 En JavaScript

```jsx
let comida = "arroz";

if (comida === "arroz") {
  console.log("¡Es la mejor comida!");
} else if (comida === "fruta") {
  console.log("No es mi favorita 😅");
} else {
  console.log("No la conozco 😄");
}

```

### ☕ En Java

```java
String comida = "arroz";

if (comida.equals("arroz")) {
    System.out.println("¡Es la mejor comida!");
} else if (comida.equals("fruta")) {
    System.out.println("No es mi favorita 😅");
} else {
    System.out.println("No la conozco 😄");
}

```

💬 *“En JavaScript usamos `===` para comparar textos; en Java usamos `.equals()`.”*

## ⚙️ **Tema 4: Uniendo todo — FizzBuzz**

### 1️⃣ Descripción técnica

FizzBuzz combina **condiciones y bucles**:

- Si el número es divisible por 3 → “Fizz”
- Si es divisible por 5 → “Buzz”
- Si es divisible por ambos → “FizzBuzz”
- En otro caso → muestra el número

---

### 2️⃣ Explicación

💬 *“Es el clásico ejercicio lógico para comprobar si entendemos las condiciones y los bucles.”*

💬 *“Vamos a hacerlo en JavaScript de dos maneras.”*

---

### 3️⃣ Ejemplo

### 🟨 **Versión 1 — Tradicional (un número)**

```jsx
let num = 15;

if (num % 3 === 0 && num % 5 === 0) {
  console.log("FizzBuzz");
} else if (num % 3 === 0) {
  console.log("Fizz");
} else if (num % 5 === 0) {
  console.log("Buzz");
} else {
  console.log(num);
}

```

### 🟨 **Versión 2 — Con bucle (1 a 20)**

```jsx
for (let num = 1; num <= 20; num++) {
  if (num % 3 === 0 && num % 5 === 0) {
    console.log("FizzBuzz");
  } else if (num % 3 === 0) {
    console.log("Fizz");
  } else if (num % 5 === 0) {
    console.log("Buzz");
  } else {
    console.log(num);
  }
}

```

---

💬 *“Este código evalúa un número a la vez, sin repetición.”*

💬 *“Aquí aplicamos todo lo aprendido:
bucle, condiciones, comparaciones y operadores.”*
💬 *“Si entendemos este código, ya dominamos el control de flujo básico.”*

Los condicionales controlan los caminos lógicos.

## 🧩 **BONUS: Switch (JavaScript) vs Switch (Java)**

### 🔹 Contexto

En JavaScript y Java existe `switch`, que sirve para tomar decisiones con múltiples casos.

---

### 🔹 Sintaxis básica en JavaScript

```jsx
let comida = "fruta";

switch (comida) {
  case "arroz":
    console.log("¡Es la mejor comida!");
    break;
  case "fruta":
    console.log("No es mi favorita 😅");
    break;
  case "verdura":
    console.log("Podría ser saludable 🥦");
    break;
  default:
    console.log("No la conozco 😄");
}

```

---

### 🔹 Sintaxis básica en Java

```java
String comida = "fruta";

switch (comida) {
    case "arroz":
        System.out.println("¡Es la mejor comida!");
        break;
    case "fruta":
        System.out.println("No es mi favorita 😅");
        break;
    case "verdura":
        System.out.println("Podría ser saludable 🥦");
        break;
    default:
        System.out.println("No la conozco 😄");
}

```

---

### 🔹 Diferencias clave

- En ambos lenguajes, **`break`** evita que se ejecuten los casos siguientes (fall-through).
- JavaScript permite `switch` con cualquier expresión, pero hay que cuidar las comparaciones (son estrictas por caso).
- Java es tipado estático y el compilador ayuda a detectar errores antes de ejecutar.
