# 🖱️ Peripherals (Periféricos)


## 🎯 Objetivo

<aside>

Entender cómo los periféricos permiten interactuar con el sistema y cómo esto se conecta con el desarrollo de software.

</aside>

---

## 🧠 ¿Qué es un periférico?

Un periférico es cualquier dispositivo que permite interactuar con un computador.

👉 Son la forma en que:

- Entregas información (input)
- Recibes información (output)

---

## 🧩 Tipos de periféricos

### 🔽 Input (Entrada)

| Dispositivo | Función | Relación con desarrollo |
| --- | --- | --- |
| ⌨️ Teclado | Escribir | Código, comandos |
| 🖱️ Mouse | Navegar | IDE, UI |
| 🎤 Micrófono | Voz | IA, comandos de voz |
| 📷 Cámara | Imagen | Reconocimiento facial |

---

### 🔼 Output (Salida)

| Dispositivo | Función | Relación con desarrollo |
| --- | --- | --- |
| 🖥️ Monitor | Visualización | UI / UX |
| 🔊 Audio | Sonido | Apps multimedia |
| 🖨️ Impresora | Físico | Sistemas legacy |

---

## 🔥 Concepto clave

👉 **Los periféricos son la interfaz entre tú y la máquina**

---

## 💡 Analogía clave (MUY IMPORTANTE)

👉 Un periférico es como una API… pero física

| Mundo físico | Mundo desarrollo |
| --- | --- |
| Teclado | Input API |
| Pantalla | Output API |
| Mouse | Eventos |
| Micrófono | Input stream |

---

## 🧠 Traducción al mundo developer

- La terminal = interfaz directa con el sistema
- El IDE = interfaz avanzada
- Un botón en frontend = input del usuario
- Un render en pantalla = output

---

## 🖥️ Ejemplo real

Cuando haces:

```
ls
```

- Tú escribes → teclado (input)
- El sistema responde → pantalla (output)

---

## 🔄 Flujo básico

```mermaid
flowchart LR
Usuario --> Teclado
Teclado --> Sistema
Sistema --> Pantalla
Pantalla --> Usuario
```

---

## ⚠️ Problemas comunes

- “No funciona mi app” → era UI, no backend
- “No responde el sistema” → input no detectado
- “Se ve mal” → problema de output (UI/UX)

---

## 🧠 Concepto clave final
👉 Todo software existe para interactuar con usuarios o sistemas
