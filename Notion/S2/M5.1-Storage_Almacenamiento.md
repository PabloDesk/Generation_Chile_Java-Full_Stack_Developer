# 💾 Storage (Almacenamiento)


## 🎯 Objetivo

<aside>

Entender **dónde vive tu código, tus datos y tu aplicación**, y cómo eso impacta directamente en el desarrollo.

</aside>

---

## 🧠 ¿Qué es el almacenamiento?

El almacenamiento es el lugar donde se guardan los datos de forma persistente.

👉 Es donde viven:

- Tu código
- Tus proyectos
- Tus archivos
- Bases de datos
- Logs

---

## 🧩 Tipos de almacenamiento

| Tipo | Característica | Ejemplo | Impacto en desarrollo |
| --- | --- | --- | --- |
| 🐢 HDD | Lento, mecánico | PC antiguos | Apps más lentas |
| ⚡ SSD | Rápido | Laptops actuales | Mejor rendimiento |
| 🚀 NVMe | Muy rápido | Equipos modernos | Ideal para backend y DB |

---

## ⚡ RAM vs Disco (CLAVE)

| Característica | RAM 🧠 | Disco 💾 |
| --- | --- | --- |
| Persistencia | ❌ No | ✅ Sí |
| Velocidad | Muy rápida | Más lenta |
| Uso | Apps en ejecución | Archivos guardados |

---

## 🔥 Concepto clave

👉 **Tu aplicación vive en RAM, pero tus datos viven en disco**

---

## 🧪 Ejemplo real

```jsx
npm start
```

- Tu app se carga en RAM
- Pero el código viene del disco

---

## 💥 ¿Qué pasa si apagas el computador?

- RAM → se borra ❌
- Disco → se mantiene ✅

---

## 🧠 Traducción al mundo developer

| Concepto técnico | En desarrollo |
| --- | --- |
| RAM | Tu backend corriendo |
| Disco | Tu proyecto guardado |
| Storage | Base de datos |
| I/O | Lectura/escritura de archivos |

---

## 🐳 Storage en el mundo real

### Docker

- Usa capas de almacenamiento
- Volúmenes → guardan datos persistentes

### Backend

- Logs se guardan en disco
- Archivos subidos → almacenamiento

### Bases de datos

- Siempre necesitan persistencia

---

## ⚠️ Problemas comunes (muy reales)

- “Mi app está lenta” → disco lento
- “Se perdió todo” → no había persistencia
- “Funciona local pero no en server” → diferencia de storage

---

## 🧠 Concepto clave final

👉 Sin almacenamiento, no existe software útil
