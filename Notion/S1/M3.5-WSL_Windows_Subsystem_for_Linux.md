# WSL (Windows Subsystem for Linux)

## 1️⃣ Sobre WSL

**WSL (Windows Subsystem for Linux)** es una funcionalidad desarrollada por Microsoft que permite **ejecutar distribuciones de Linux directamente en Windows**, sin necesidad de usar una máquina virtual ni un sistema de arranque dual.

WSL crea una **capa de compatibilidad** entre Windows y Linux, lo que permite ejecutar herramientas, comandos y entornos de desarrollo nativos de Linux (como `bash`, `apt`, `ssh`, `git`, o `python`) dentro del propio Windows.

Esto facilita enormemente el trabajo de desarrolladores, administradores de sistemas y profesionales DevOps que necesitan usar ambos entornos de manera integrada.

A diferencia de una máquina virtual, **WSL no requiere virtualización completa**, lo que significa que utiliza menos recursos, inicia más rápido y ofrece una integración directa con el sistema de archivos de Windows.

## 2️⃣ Requisitos previos

- Windows 10 (versión 2004 o superior) o **Windows 11**.
- **Virtualización activada** en BIOS/UEFI.
- Permisos de administrador.
- Conexión a Internet.

---

## 3️⃣ Instalación rápida (recomendada)

1. Abre **PowerShell** o **Windows Terminal** como administrador.
2. Ejecuta el siguiente comando:
    
    ```bash
    wsl --install
    ```
    
    Este comando:
    
    - Activa los componentes necesarios (WSL y Virtual Machine Platform).
    - Descarga el **kernel Linux**.
    - Instala **Ubuntu** por defecto.
    - Configura **WSL 2** como versión predeterminada.
3. Reinicia tu equipo.
4. Al reiniciar, crea tu usuario y contraseña de Linux.

---

## 4️⃣ ¿Por qué usar WSL?

WSL es una herramienta clave para desarrolladores y equipos técnicos que trabajan en entornos mixtos (Windows y Linux).

Algunas de sus principales ventajas son:

- 🧩 **Compatibilidad total con Linux:** ejecuta binarios ELF64 de Linux sin modificaciones.
- ⚙️ **Integración con Windows:** puedes acceder a archivos de Windows desde Linux y viceversa.
- ⚡ **Alto rendimiento:** arranca en segundos, sin los recursos que consume una máquina virtual.
- 🧱 **Ideal para desarrollo full stack:** puedes usar Node.js, Python, Java, Docker, Git o Kubernetes directamente desde tu subsistema Linux.
- ☁️ **Perfecto para DevOps:** ideal para entornos de CI/CD, scripts de automatización, y despliegues cloud.

---

## 5️⃣ Versiones de WSL

| Versión | Características Principales |
| --- | --- |
| **WSL 1** | Traduce las llamadas del kernel Linux a Windows. Es más rápido en acceso a archivos de Windows, pero no ejecuta contenedores. |
| **WSL 2** | Ejecuta un **kernel Linux real** dentro de una máquina ligera optimizada por Microsoft. Ofrece compatibilidad total con Docker, mejor rendimiento de sistema de archivos y soporte completo para herramientas de red. |

> 💡 En la actualidad, WSL 2 es la versión recomendada por Microsoft.
> 
> 
> Si instalas WSL desde cero, se instalará automáticamente esta versión.
>

## 6️⃣ Instalación manual (Alternativa)

Si `wsl --install` no está disponible, sigue estos pasos:

1. **Habilitar características:**
    
    ```bash
    dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
    dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
    ```
    
2. **Reinicia** el sistema.
3. **Descarga e instala** el kernel desde: https://aka.ms/wsl2kernel
4. Define WSL 2 como predeterminado:
    
    ```bash
    wsl --set-default-version 2
    ```
    
5. **Instala una distribución** (por ejemplo, Ubuntu):
    
    ```bash
    wsl --install -d Ubuntu
    ```
    

---

### 🧠 Verificar instalación

Ejecuta:

```bash
wsl --status
```

Y asegúrate de que la versión por defecto sea la 2.

Puedes comprobar tus distribuciones instaladas con:

```bash
wsl --list --verbose
```

## 7️⃣ Comandos Básicos de WSL

| Comando | Descripción |
| --- | --- |
| `wsl` | Abre la terminal de Linux por defecto. |
| `wsl --list` o `wsl -l` | Muestra las distribuciones instaladas. |
| `wsl --list --online` | Muestra todas las distribuciones disponibles para instalar. |
| `wsl --install -d <distro>` | Instala una distribución específica (ej: Ubuntu). |
| `wsl --set-version <distro> 2` | Cambia la versión de WSL usada por una distribución. |
| `wsl --set-default-version 2` | Establece WSL 2 como versión predeterminada. |
| `wsl --unregister <distro>` | Elimina una distribución de Linux. |
| `wsl --terminate <distro>` | Detiene una distribución en ejecución. |
| `wsl --shutdown` | Cierra todas las distribuciones y detiene el kernel de WSL. |
| `explorer.exe .` | Abre el explorador de archivos de Windows en el directorio actual de Linux. |
| `code .` | Abre Visual Studio Code en el entorno actual (requiere instalación de VS Code y extensión WSL). |
