# Introducción a Linux

## 🧠 1. ¿Qué es un Sistema Operativo (SO)?

Un **Sistema Operativo (SO)** es el software principal que **gestiona los recursos de hardware y software** de un computador y proporciona una interfaz entre el usuario y la máquina.

Permite ejecutar programas, coordinar tareas y asegurar que los procesos usen eficientemente la CPU, memoria, almacenamiento y dispositivos.

### Funciones principales

- **Gestión de procesos:** controla la ejecución de programas y su prioridad.
- **Gestión de memoria:** asigna y libera recursos según lo requieran los programas.
- **Gestión de archivos:** organiza la información en sistemas de archivos (ext4, NTFS, APFS, etc.).
- **Gestión de dispositivos:** controla periféricos mediante drivers.
- **Interfaz con el usuario:** puede ser gráfica (GUI) o de línea de comandos (CLI).

## ⚙️ 3. ¿Qué es Linux?

**Linux no es un sistema operativo completo por sí mismo**, sino un **núcleo (kernel)**.

El **kernel** es el componente central del sistema operativo: **gestiona la comunicación entre el hardware y el software**.

> 🔹 Linux = Kernel (núcleo)
> 
> 
> 🔹 **Distribución Linux = Kernel + herramientas + entorno gráfico + utilidades + gestor de paquetes**
> 

### Ejemplo

Ubuntu, Fedora, Debian y Red Hat son **distribuciones de Linux**; todas usan el mismo núcleo, pero incluyen diferentes paquetes, entornos y configuraciones.

## 🧩 4. Conceptos Fundamentales

| Concepto | Definición Técnica |
| --- | --- |
| **Kernel (Núcleo)** | Es la parte central del sistema operativo. Controla los recursos del hardware y facilita la comunicación con el software. |
| **Distribución (Distro)** | Versión completa del sistema operativo basada en el kernel Linux, con su propio gestor de paquetes, entorno de escritorio y herramientas (ej: Ubuntu, Fedora, Arch, Debian). |
| **Shell** | Programa que interpreta comandos del usuario (Bash, Zsh, Fish). Es la capa entre el kernel y el usuario. |
| **Terminal** | Interfaz (CLI) que permite comunicarse con el shell y ejecutar comandos. |
| **Open Source** | Filosofía y modelo de licencia que permite acceder, modificar y redistribuir libremente el código fuente. |

---

## 🧠 5. ¿Qué significa “Open Source”?

El **software de código abierto** (Open Source) es aquel cuyo **código fuente puede ser estudiado, modificado y redistribuido libremente** bajo licencias que garantizan esos derechos (como la GPL).

### Diferencias con el software privativo

| Aspecto | Código Abierto | Privativo |
| --- | --- | --- |
| Acceso al código | Libre y público | Cerrado y restringido |
| Licencia | Gratuita (GPL, MIT, Apache, BSD) | De pago o bajo suscripción |
| Modificación | Permitida | Prohibida |
| Ejemplos | Linux, Firefox, LibreOffice | Windows, macOS, Microsoft Office |

> 💡 Puedes crear tu propia distribución de Linux partiendo del kernel, agregando tus propios programas y personalizaciones.
> 
> 
> Esto es legal y posible gracias a las licencias de software libre.
>

## 🏗️ 6. ¿Puedo ganar dinero con software libre?

Sí. Aunque el software sea gratuito, **el modelo de negocio se basa en los servicios**, no en la venta del código:

- Soporte técnico (ej: Red Hat Enterprise Linux)
- Consultoría y personalización
- Hosting, mantenimiento o capacitación
- Integración de soluciones empresariales

## 🖥️ 7. Diferencias entre un PC tradicional y un Servidor

| Característica | PC de Escritorio | Servidor |
| --- | --- | --- |
| **Propósito** | Uso personal (ofimática, diseño, navegación, juegos) | Ofrecer servicios a otros equipos o usuarios |
| **Usuarios** | Uno o pocos | Decenas o miles simultáneamente |
| **Interfaz** | Con monitor, teclado y GUI | Usualmente sin interfaz gráfica (CLI) |
| **Gestión** | Local | Remota (SSH, Web Admin, etc.) |
| **Hardware** | Optimizado para uso individual | Diseñado para operar 24/7 con redundancia y alto rendimiento |

---

## 🌐 8. Tipos de Servidores Comunes

| Tipo de Servidor | Función | Ejemplo / Software |
| --- | --- | --- |
| **Servidor Web** | Aloja sitios web y entrega páginas a los navegadores. | Apache, Nginx |
| **Servidor de Base de Datos** | Gestiona y almacena información estructurada. | MySQL, PostgreSQL |
| **Servidor de Archivos** | Permite compartir archivos en red. | Samba, NFS |
| **Servidor de Correo** | Envía y recibe correos electrónicos. | Postfix, Sendmail |
| **Servidor de Aplicaciones** | Ejecuta software empresarial o servicios backend. | Tomcat, Node.js |
| **Servidor DNS / DHCP** | Gestiona nombres de dominio y asigna direcciones IP. | BIND, isc-dhcpd |
| **Servidor Virtual o Cloud** | Aloja máquinas virtuales o contenedores. | AWS EC2, Azure, Proxmox |

## 🚀 9. La Importancia de Linux en el Mundo Actual

- Domina más del **96 % de los servidores** del mundo.
- Es la base de **Android**, **Chrome OS**, **routers**, **smart TV**, **IoT**, y **supercomputadoras**.
- Es el **estándar en DevOps, ciberseguridad, cloud computing y data science**.
- Gracias a su estabilidad, seguridad y flexibilidad, **es el preferido para entornos empresariales y de desarrollo**.
