# 🧠 **1. Qué es la Lógica de Programación**

> Es la habilidad de organizar instrucciones de forma coherente y ordenada para lograr un objetivo.
> 

Es la base de **todo lenguaje de programación**.

Da lo mismo si programas en:

- JavaScript
- Python
- Java
- C#
- Go

La lógica **siempre es la misma**.

<aside>
💡

> 
> 
> 
> La lógica de programación es universal.
> 
> Solo cambia la forma en que cada lenguaje escribe las instrucciones.
> 
</aside>

---

## 📌 Claves de la lógica

- Está presente en **todo lenguaje de programación**
- Se basa en **patrones universales**
- Permite **resolver problemas de forma estructurada**

Patrones comunes:

- Variables
- Condicionales
- Ciclos
- Funciones

---

# ☕ Ejemplo cotidiano de lógica

Imagina preparar café.

### Pasos

1️⃣ Calentar agua

2️⃣ Poner café en la taza

3️⃣ Verter agua

4️⃣ Revolver

5️⃣ Servir

Cada paso ocurre **en orden**.

<aside>
💡

> 
> 
> 
> Un programa funciona exactamente igual.
> 
</aside>

---

# 🔄 Patrón universal de un programa

Todo programa sigue esta estructura:

```
Entrada → Proceso → Salida
```

| Etapa | Qué significa |
| --- | --- |
| Entrada | Datos que recibe el programa |
| Proceso | Operaciones que se realizan |
| Salida | Resultado obtenido |

---

## Ejemplo simple

**Problema:** calcular la edad de una persona

| Paso | Acción |
| --- | --- |
| Entrada | Año actual, año de nacimiento |
| Proceso | Restar nacimiento al año actual |
| Salida | Edad calculada |

# 🔢 **2. Algoritmos**

<aside>

> Un algoritmo es una secuencia de pasos ordenados que permiten resolver un problema.
> 
</aside>

Los algoritmos están presentes en:

- programas
- aplicaciones
- inteligencia artificial
- motores de búsqueda
- redes sociales

---

## 📋 Características de un buen algoritmo

| # | Característica | Descripción |
| --- | --- | --- |
| 1️⃣ | Preciso | Cada paso es claro |
| 2️⃣ | Definido | Siempre produce el mismo resultado |
| 3️⃣ | Finito | Tiene un final |
| 4️⃣ | Legible | Es fácil de entender |
| 5️⃣ | Concreto | Resuelve un problema específico |

---

# ⚙️ Estructura de un algoritmo

Un algoritmo normalmente se divide en:

### 📥 Entrada

Datos que recibe el programa.

### ⚙️ Proceso

Operaciones que realiza.

### 📤 Salida

Resultado obtenido.

---

# 💬 **3. Pseudocódigo**

<aside>

> El pseudocódigo es una forma de escribir algoritmos usando lenguaje natural.
> 
</aside>

No usa sintaxis de programación.

Su objetivo es **explicar la lógica de forma clara**.

---

## 🧩 Ejemplo en pseudocódigo

```
Inicio
Leer año_actual
Leer año_nacimiento
edad ← año_actual - año_nacimiento
Escribir "La edad es:", edad
Fin
```

---

## 🧠 Beneficios del pseudocódigo

- Permite **pensar antes de programar**
- Facilita la **comunicación entre desarrolladores**
- Ayuda a **detectar errores antes de escribir código**

<aside>
💡

> 
> 
> 
> El pseudocódigo es el puente entre la idea y el programa.
> 
</aside>

---

# 🔗 De la lógica al código real

Pseudocódigo:

```
Inicio
Leer nombre
Mostrar "Hola " + nombre
Fin
```

---

### Versión en JavaScript

```jsx
let nombre = prompt("¿Cómo te llamas?");
console.log("Hola "+nombre);
```

Aquí ocurre lo mismo:

| Lógica | JavaScript |
| --- | --- |
| Leer nombre | `prompt()` |
| Mostrar mensaje | `console.log()` |

## 💬 **Mantras del programador lógico**

<aside>

Aprender programación no significa memorizar código.

Significa **aprender a pensar de forma estructurada para resolver problemas**.

El lenguaje es solo una herramienta.

</aside>

- 🧭 Piensa antes de escribir código
- 💡 Cada paso importa
- 🔍 Divide el problema en partes pequeñas
- 🧱 *Construye primero la estructura, luego el código.*
- 🤝 *Usa el pseudocódigo para comunicar ideas.*
- 🚀 *Un algoritmo bien pensado vale más que mil líneas mal escritas.*
