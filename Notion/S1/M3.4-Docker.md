## 1️⃣ Sobre Docker

Docker es una plataforma de código abierto que permite a los desarrolladores **crear, empaquetar, distribuir y ejecutar aplicaciones dentro de contenedores**. Un contenedor es una unidad estándar de software que empaqueta el código y todas sus dependencias, lo que garantiza que la aplicación se ejecute de manera rápida y confiable en cualquier entorno informático.

A diferencia de las máquinas virtuales, los contenedores no incluyen un sistema operativo completo; solo empaquetan las librerías y configuraciones necesarias, haciéndolos mucho más **ligeros, rápidos y eficientes**. Esto facilita el desarrollo, las pruebas y el despliegue de aplicaciones, asegurando que funcionen de la misma manera en el entorno de un desarrollador, en un servidor de pruebas o en producción

## 2️⃣ Guía de instalación de Docker

Para instalar Docker, sigue los pasos correspondientes a tu sistema operativo. La forma más común y recomendada para Windows y macOS es a través de **Docker Desktop**.

<aside>

**Consejo importante:** Docker Desktop es la herramienta oficial de Docker para entornos de desarrollo en Windows y macOS. No solo instala el motor de Docker (Docker Engine), sino que también proporciona una interfaz gráfica de usuario (GUI) para gestionar contenedores, imágenes y volúmenes, simplificando enormemente el flujo de trabajo.

</aside>

### **Windows**

1. Ve al sitio oficial y descarga [**Docker Desktop for Windows**](https://www.docker.com/products/docker-desktop/).
2. Ejecuta el instalador y sigue las instrucciones. Es posible que requiera activar WSL 2 (Windows Subsystem for Linux), lo cual el instalador puede hacer por ti.
3. Reinicia tu sistema si se te solicita.
4. Verifica la instalación abriendo la terminal (PowerShell o CMD) y ejecutando: `docker --version`.

### macOS

1. **Opción 1 (Recomendada):** Descarga [**Docker Desktop for Mac**](https://www.docker.com/products/docker-desktop/) desde el sitio oficial.
2. **Opción 2 (Homebrew):** `brew install --cask docker`.
3. Arrastra el ícono de Docker a tu carpeta de Aplicaciones e inícialo.
4. Verifica la instalación abriendo la terminal y ejecutando: `docker --version`.

### Linux (Ubuntu/Debian)

Aunque Docker Desktop está disponible para Linux, la instalación a través de la terminal sigue siendo muy común.

1. Actualiza tus repositorios: `sudo apt-get update`.
2. Instala los paquetes necesarios para permitir a `apt` usar un repositorio sobre HTTPS:
    
    ```bash
    sudo apt-get install \
        ca-certificates \
        curl \
        gnupg
    ```
    
3. Añade la clave GPG oficial de Docker:
    
    ```bash
    sudo install -m 0755 -d /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    sudo chmod a+r /etc/apt/keyrings/docker.gpg
    ```
    
4. Configura el repositorio de Docker:
    
    ```bash
    echo \
      "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
      $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
      sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    ```
    
5. Instala Docker Engine:
    
    ```bash
    sudo apt-get update
    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
    ```
    
6. Verifica la instalación ejecutando el contenedor de prueba: `sudo docker run hello-world`.
    
    <aside>
    
    **Consejo importante:** En Linux, después de la instalación, es posible que necesites ejecutar los comandos de Docker con `sudo`.  Para evitar esto, puedes añadir tu usuario al grupo `Docker`:
    
    ```bash
    sudo usermod -aG docker $USER
    ```
    
    Después de ejecutar este comando, deberás cerrar sesión y volver a iniciarla (o reiniciar el sistema) para que los cambios surtan efecto.
    
    </aside>
    

## 3️⃣ **Conceptos y Comandos Básicos**

Para empezar a trabajar con Docker, es fundamental entender algunos conceptos clave.

- **Imagen (Image):** Una plantilla de solo lectura con instrucciones para crear un contenedor. Las imágenes contienen el código de la aplicación, las librerías, las herramientas y otras dependencias.
- **Contenedor (Container):** Una instancia en ejecución de una imagen. Es el entorno aislado donde vive y se ejecuta tu aplicación.
- **Dockerfile:** Un archivo de texto que contiene los comandos necesarios para construir una imagen de Docker.
- **Docker Hub:** Un registro (como GitHub, pero para imágenes de Docker) donde puedes encontrar y compartir imágenes de contenedores.

**Comandos básicos de Docker:**

- **Listar imágenes locales:** `docker images`
- **Descargar una imagen desde Docker Hub:** `docker pull <nombre-imagen>:<tag>`
    - Ejemplo: `docker pull node:18`
- **Ejecutar un contenedor a partir de una imagen:** `docker run [opciones] <nombre-imagen>`
    - Ejemplo interactivo: `docker run -it ubuntu bash` (inicia una terminal dentro de un contenedor de Ubuntu).
    - Ejemplo en segundo plano: `docker run -d -p 8080:80 nginx` (ejecuta un servidor web Nginx y mapea el puerto 80 del contenedor al 8080 de tu máquina).
- **Listar contenedores en ejecución:** `docker ps`
- **Listar todos los contenedores (en ejecución y detenidos):** `docker ps -a`
- **Detener un contenedor en ejecución:** `docker stop <id-del-contenedor-o-nombre>`
- **Eliminar un contenedor detenido:** `docker rm <id-del-contenedor-o-nombre>`
- **Ver los logs (registros) de un contenedor:** `docker logs <id-del-contenedor-o-nombre>`

<aside>
Docker es una herramienta extremadamente potente que revoluciona la forma en que se desarrollan y despliegan las aplicaciones. Aprender a usarlo es una habilidad clave para cualquier desarrollador moderno.
