# Git
## 1️⃣ Sobre Git

Git es un sistema de control de versiones distribuido que permite registrar los cambios de tu código a lo largo del tiempo, colaborar con otras personas y mantener múltiples líneas de desarrollo en paralelo. Con Git puedes:

- Guardar “fotografías” del estado de tu proyecto llamadas commits.
- Crear ramas para desarrollar nuevas características sin romper la rama principal.
- Integrarte con servicios como GitHub, GitLab o Bitbucket para colaborar y revisar código.

## 2️⃣ Guía de instalación de Git

Sigue los pasos según tu sistema operativo. Al finalizar, verifica con: `git --version`.

### Windows

1. Descarga el instalador desde [gitforwindows.org](http://gitforwindows.org) o desde el [sitio oficial](https://git-scm.com/downloads).
2. Ejecuta el instalador y acepta las opciones por defecto a menos que necesites algo específico.
3. Abre “Git Bash” o PowerShell y ejecuta: `git --version`.

<aside>

>Consejo: para actualizar en Windows usa el comando `winget upgrade --id Git.Git`

</aside>

<aside>

>Consejo: Durante la instalación, deja “Use Git from the command line” y “Checkout as-is, commit Unix-style line endings” salvo que tu equipo indique otra cosa.

</aside>

### macOS

- Opción 1: Con Homebrew:
    
    ```bash
    brew install git
    ```
    
- Opción 2: Instalar Xcode Command Line Tools y aceptar la instalación de Git al ejecutar `git` por primera vez.
    
    ```bash
    xcode-select --install
    ```
    
- Opción 3: Descarga el instalador para macOS desde el [sitio oficial](https://git-scm.com/downloads).

Verifica:

<aside>
💡

>Al finalizar cualquier instalación, confirma con: `git --version`.

</aside>

### Linux

Según la distribución, utiliza tu gestor de paquetes:

- Debian/Ubuntu:
    
    ```bash
    sudo apt update
    sudo apt install git
    ```
    
- Fedora:
    
    ```bash
    sudo dnf install git
    ```
    
- Arch Linux:
    
    ```bash
    sudo pacman -S git
    ```
    

Verifica:

```bash
git --version
```

## 3️⃣ Configuración inicial recomendada

Configura tu identidad y editor por defecto.

```jsx
git config --global [user.name](http://user.name) "Tu Nombre"
git config --global [user.email](http://user.email) "[tu.email@ejemplo.com](mailto:tu.email@ejemplo.com)"
# Editor (ejemplo con VS Code)
git config --global core.editor "code --wait"
# Colorear la salida en la terminal
git config --global color.ui auto
```
Para resetear ocupar:
```jsx
git config --global --unset-all user.name
git config --global --unset-all user.email
```
Para resetear ocupar:
```jsx
git config --global user.name "Jose Flores"
git config --global user.email "joseflores64@gmail.com"
```

Crea un archivo `.gitignore` para excluir archivos temporales o credenciales. Consulta plantillas en github/gitignore.

## 4️⃣ Conceptos clave

- Repositorio: Carpeta administrada por Git que contiene tu historial.
- Commit: Punto en el tiempo con cambios y mensaje descriptivo.
- Rama (branch): Línea de trabajo independiente.
- Remoto (remote): Copia del repositorio alojada en un servidor (por ejemplo, GitHub).

---

## 5️⃣ Flujo básico y comandos útiles

Inicializa, registra cambios y comparte tu trabajo.

### Iniciar o clonar

```bash
# Iniciar en carpeta existente
cd mi-proyecto
git init

# Clonar un repositorio existente
git clone https://ejemplo.com/usuario/proyecto.git
```

### Estado, añadir y confirmar

```bash
git status
# Añadir archivos al área de preparación
git add archivo.txt
# Añadir todo
git add .
# Crear commit con mensaje
git commit -m "Descripción breve del cambio"
```

### Ramas

```bash
# Crear y cambiar a una rama
git checkout -b feature/nueva-funcionalidad
# (o con switch)
git switch -c feature/nueva-funcionalidad
# Volver a main
git switch main
```

### Integración y sincronización

```bash
# Añadir remoto
git remote add origin https://ejemplo.com/usuario/proyecto.git
# Traer cambios del remoto
git pull origin main
# Enviar tus commits
git push -u origin main
```

### Revisión del historial

```bash
git log --oneline --graph --decorate --all
```

<aside>

Practicar con repositorios de prueba es la forma más rápida de dominar Git. Empieza por flujos pequeños: crear una rama, hacer uno o dos commits y abrir un pull request en tu forja favorita.

</aside>
