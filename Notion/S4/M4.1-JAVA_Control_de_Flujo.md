# **JAVA - Control de Flujo**


# 🎯 **1. Introducción al Control de Flujo**

El **control de flujo** en Java permite decidir *qué código se ejecuta*, *cuántas veces* y *en qué orden*.

Es la base para que un programa pueda “pensar”, evaluar condiciones, tomar decisiones y repetir acciones.

Java ofrece tres grandes familias de control de flujo:

1. **Condicionales** → if / else / switch
2. **Bucles** → while / do-while / for
3. **Control interno** → break, continue (veremos más adelante)

En esta guía veremos **todas las formas esenciales**, con ejemplos aptos para estudiantes de bootcamp.

---

# 🧩 **2. Condicionales (if / else)**

Se usan cuando el programa debe elegir entre distintos caminos según una condición.

---

## 🟦 **2.1. if**

Ejecuta un bloque si la condición es verdadera.

```java
int edad = 18;

if (edad >= 18) {
    System.out.println("Puedes entrar");
}
```

---

## 🟩 **2.2. if / else**

```java
if (edad >= 18) {
    System.out.println("Bienvenido");
} else {
    System.out.println("No puedes ingresar");
}
```

---

## 🟧 **2.3. if / else if / else**

Para múltiples condiciones:

```java
if (hora < 12) {
    System.out.println("Buenos días");
} else if (hora < 18) {
    System.out.println("Buenas tardes");
} else {
    System.out.println("Buenas noches");
}
```

---

## ⚠️ Buenas Prácticas con `if`

✔ Mantener condiciones claras

✔ Primero validar casos especiales

✔ Evitar if anidados muy profundos

❌ No escribir condiciones demasiado largas:

```java
if (a > b && c < d || e == f && g != h) // ❌ confuso
```

✔ Mejor dividir en variables intermedias:

```java
boolean esMayor = a > b;
boolean esMenor = c < d;

if (esMayor && esMenor) { ... }
```

# 🧩 **3. Switch (incluye versión moderna de Java 14 → 21)**

`switch` permite evaluar un valor contra varios casos.

Es útil cuando hay muchas condiciones basadas en igualdad.

---

## 🟦 3.1 Switch tradicional

```java
int opcion = 2;

switch (opcion) {
    case 1:
        System.out.println("Nuevo Juego");
        break;
    case 2:
        System.out.println("Cargar Partida");
        break;
    case 3:
        System.out.println("Salir");
        break;
    default:
        System.out.println("Opción inválida");
}
```

---

## 🟧 3.2 Switch moderno (Java 14 → 21)

Más claro, sin necesidad de `break`.

```java
String rol = "admin";

String mensaje = switch (rol) {
    case "admin" -> "Acceso total";
    case "user" -> "Acceso limitado";
    default -> "Sin acceso";
};

System.out.println(mensaje);
```

✔ Más breve

✔ Más seguro

✔ Permite asignar valores directamente

# 🔁 **4. Bucles (Looping)**

Los bucles permiten repetir acciones.

Java tiene:

| Bucle | ¿Cuándo usarlo? |
| --- | --- |
| **while** | Repetir mientras una condición sea verdadera |
| **do-while** | Repetir al menos 1 vez |
| **for** | Repetir un número exacto de veces |
| **foreach** | Recorrer colecciones o arreglos |

*(foreach se ve formalmente más adelante, pero es útil mencionarlo aquí)*

---

## 🟦 4.1 while

```java
int i = 1;

while (i <= 5) {
    System.out.println(i);
    i++;
}
```

---

## 🟧 4.2 do-while

Siempre se ejecuta al menos una vez.

```java
int x = 0;

do {
    System.out.println("Esto se ejecuta mínimo una vez");
    x++;
} while (x < 0);
```

---

## 🟥 4.3 for

```java
for (int i = 1; i <= 10; i++) {
    System.out.println("Número: " + i);
}
```

---

## ⚠️ Buenas prácticas de loops

✔ Evitar loops infinitos

✔ Mantener límites claros

✔ Usar nombres descriptivos si no es un loop pequeño

❌ Esto es crítico de NO hacer:

```java
while (true) { } // infinito sin control
```

---

# 🧮 **5. Combinación de Control de Flujo**

Ejemplo que combina condiciones + loops:

```java
int vida = 30;

while (vida > 0) {
    if (vida <= 10) {
        System.out.println("¡Peligro!");
    }

    System.out.println("Vida: " + vida);
    vida -= 5;
}
```

---

# 🚫 **6. Errores Comunes en Control de Flujo**

| Error | Explicación | Corrección |
| --- | --- | --- |
| Condiciones con `=` en vez de `==` | `=` asigna | usar `==` |
| Olvidar el incremento del while | loop infinito | sumar o restar dentro del loop |
| else sin if | estructura inválida | siempre acompañar |
| switch sin default | casos no cubiertos | agregar `default` |

---

# 💡 **7. Buenas Prácticas**

### ✔ Estructurar de lo general a lo específico

Primero los casos simples, luego los complejos.

### ✔ Evitar duplicación de código

Si tienes varios if que hacen lo mismo → algo está mal.

### ✔ Mantener indentación clara

Un mal identado confunde incluso a desarrolladores senior.

### ✔ Usar switch moderno cuando sea más legible

Especialmente en menús o roles.

# 🧪 **8. Ejemplos**

## Ejemplo 1 — Validación simple

```java
int edad = 20;

if (edad >= 18) {
    System.out.println("Mayor de edad");
} else {
    System.out.println("Menor de edad");
}
```

---

## Ejemplo 2 — Menú simple con switch moderno

```java
int opcion = 1;

String resultado = switch (opcion) {
    case 1 -> "Iniciando juego";
    case 2 -> "Cargando partida";
    case 3 -> "Saliendo";
    default -> "Opción no válida";
};

System.out.println(resultado);
```

---

## Ejemplo 3 — Loop con condición

```java
int contador = 1;

while (contador <= 3) {
    System.out.println("Intento #" + contador);
    contador++;
}
```
