# Ejercicio Guiado - Sistema de Gestión de Ventas

## 🎯 Objetivo general

Diseñar y construir una **base de datos relacional completa**, crear sus tablas con **llaves primarias y foráneas**, ingresar datos de prueba y responder consultas SQL que representen **preguntas reales de negocio**.

---

## 🧩 Contexto del sistema

Una empresa necesita un sistema simple para registrar:

- Clientes
- Productos
- Ventas realizadas
- Detalles de cada venta

El sistema debe permitir analizar:

- Quién compra
- Qué se vende
- Cuánto se vende
- Cuándo se vende

---

## 📐 Requerimientos de modelado

Cada grupo debe:

1. Identificar las **entidades principales**
2. Definir los **atributos** de cada entidad
3. Establecer las **relaciones**
4. Definir correctamente:
    - Llaves primarias (PK)
    - Llaves foráneas (FK)
5. Usar **IDs autoincrementales**
6. Garantizar **integridad referencial**

---

## 🧱 Entidades mínimas esperadas

El modelo debe incluir **al menos** las siguientes entidades:

### 🧍 Cliente

- Identificador único
- Nombre
- Email

### 📦 Producto

- Identificador único
- Nombre
- Precio

### 🧾 Venta

- Identificador único
- Fecha
- Cliente asociado

### 🧾 Detalle de venta

- Venta asociada
- Producto asociado
- Cantidad vendida

📌 Las relaciones deben reflejar la realidad del negocio.

---

## 🛠️ Trabajo a realizar por el grupo

### Parte 1 — Diseño

- Dibujar el **Diagrama Entidad–Relación**
- Definir cardinalidades (1:1, 1:N, N:N)
- Justificar las tablas intermedias

---

### Parte 2 — Implementación

- Crear la base de datos
- Crear todas las tablas
- Definir PK y FK
- Insertar **datos suficientes** para responder todas las consultas

---

## ❓ Consultas a resolver (SQL)

Una vez creada y poblada la base de datos, cada grupo debe responder **todas las siguientes consultas** usando SQL.

---

### 🔍 Consultas básicas

1. Mostrar **todos los clientes** registrados.
2. Mostrar **todos los productos** disponibles.
3. Mostrar **todas las ventas** realizadas.
4. Mostrar solo el **nombre y email** de los clientes.

---

### 📊 Consultas con filtros y orden

1. Mostrar los productos con precio **mayor a un valor determinado**.
2. Mostrar las ventas realizadas en una **fecha específica**.
3. Mostrar los productos ordenados por **precio de mayor a menor**.

---

### 🧮 Consultas con agregación

1. ¿Cuántos clientes existen en total?
2. ¿Cuál es el **precio promedio** de los productos?
3. ¿Cuántas ventas se han realizado en total?

---

### 🔗 Consulta con JOIN (máximo 1 JOIN)

1. Mostrar las ventas indicando:
- Nombre del cliente
- Fecha de la venta

📌 *No se permite usar más de un JOIN en esta consulta.*

---

### 🧠 Consulta “trampa” (resultado vacío)

1. Crear una consulta que **no devuelva registros**, pero que esté **correctamente escrita**.

Ejemplo de lógica esperada:

- Buscar ventas en una fecha donde no existan registros
- Buscar productos con precio fuera del rango real

👉 Se debe justificar **por qué el resultado vacío es correcto**.

---

## 📌 Restricciones del challenge

- No usar `LEFT JOIN` ni `RIGHT JOIN`
- No usar subconsultas
- No usar funciones avanzadas
- Solo SQL básico (`SELECT`, `WHERE`, `ORDER BY`, `COUNT`, `AVG`, `JOIN`)
- El diseño debe ser **coherente y consistente**

    # 🐘 Script SQL PostgreSQL — Creación de tablas
    
    ```sql
    -- =========================================
    -- 1. Crear tablas
    -- =========================================
    
    CREATE TABLE clientes (
        id_cliente INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
        nombre VARCHAR(100) NOT NULL,
        email VARCHAR(100) NOT NULL UNIQUE
    );
    
    CREATE TABLE productos (
        id_producto INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
        nombre VARCHAR(100) NOT NULL,
        precio NUMERIC(10,2) NOT NULL CHECK (precio > 0)
    );
    
    CREATE TABLE ventas (
        id_venta INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
        fecha DATE NOT NULL,
        id_cliente INT NOT NULL,
        CONSTRAINT fk_ventas_cliente
            FOREIGN KEY (id_cliente)
            REFERENCES clientes(id_cliente)
            ON DELETE RESTRICT
            ON UPDATE CASCADE
    );
    
    CREATE TABLE detalle_venta (
        id_detalle INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
        id_venta INT NOT NULL,
        id_producto INT NOT NULL,
        cantidad INT NOT NULL CHECK (cantidad > 0),
        CONSTRAINT fk_detalle_venta_venta
            FOREIGN KEY (id_venta)
            REFERENCES ventas(id_venta)
            ON DELETE RESTRICT
            ON UPDATE CASCADE,
        CONSTRAINT fk_detalle_venta_producto
            FOREIGN KEY (id_producto)
            REFERENCES productos(id_producto)
            ON DELETE RESTRICT
            ON UPDATE CASCADE
    );
    ```

    # 🔍 Consultas solicitadas
    
    ---
    
    ## 1) Consultas básicas
    
    ### 1. Mostrar todos los clientes registrados
    
    ```sql
    SELECT * FROM clientes;
    ```
    
    ### 2. Mostrar todos los productos disponibles
    
    ```sql
    SELECT * FROM productos;
    ```
    
    ### 3. Mostrar todas las ventas realizadas
    
    ```sql
    SELECT * FROM ventas;
    ```
    
    ### 4. Mostrar solo el nombre y email de los clientes
    
    ```sql
    SELECT nombre, email
    FROM clientes;
    ```
    
    ---
    
    ## 2) Consultas con filtros y orden
    
    ### 1. Mostrar los productos con precio mayor a un valor determinado
    
    Ejemplo: mayor a 40000
    
    ```sql
    SELECT *
    FROM productos
    WHERE precio > 40000;
    ```
    
    ### 2. Mostrar las ventas realizadas en una fecha específica
    
    Ejemplo: ventas del 2026-04-02
    
    ```sql
    SELECT *
    FROM ventas
    WHERE fecha = '2026-04-02';
    ```
    
    ### 3. Mostrar los productos ordenados por precio de mayor a menor
    
    ```sql
    SELECT *
    FROM productos
    ORDER BY precio DESC;
    ```
    
    ---
    
    ## 3) Consultas con agregación
    
    ### 1. ¿Cuántos clientes existen en total?
    
    ```sql
    SELECT COUNT(*) AS total_clientes
    FROM clientes;
    ```
    
    ### 2. ¿Cuál es el precio promedio de los productos?
    
    ```sql
    SELECT AVG(precio) AS precio_promedio
    FROM productos;
    ```
    
    ### 3. ¿Cuántas ventas se han realizado en total?
    
    ```sql
    SELECT COUNT(*) AS total_ventas
    FROM ventas;
    ```
    
    ---
    
    ## 4) Consulta con JOIN (máximo 1 JOIN)
    
    ### Mostrar las ventas indicando nombre del cliente y fecha de la venta
    
    ```sql
    SELECT c.nombre, v.fecha
    FROM ventas v
    JOIN clientes c ON v.id_cliente = c.id_cliente;
    ```
    
    ✅ Esta consulta usa exactamente **1 JOIN**.
    
    ---
    
    ## 5) Consulta trampa (resultado vacío)
    
    ### Buscar productos con precio mayor a 2000000
    
    ```sql
    SELECT *
    FROM productos
    WHERE precio > 2000000;
    ```
    
    ### Justificación
    
    El resultado vacío es correcto porque, según los datos insertados, **ningún producto tiene un precio superior a 2.000.000**.
    
    Por lo tanto, la consulta está bien escrita, pero no existen registros que cumplan esa condición.
