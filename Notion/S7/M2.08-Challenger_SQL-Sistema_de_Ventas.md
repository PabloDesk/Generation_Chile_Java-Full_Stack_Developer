# 🧩 Challenger SQL - Sistema de Ventas

# 🏢 Contexto del caso

Una tienda de tecnología necesita registrar la información básica de sus operaciones.

El sistema debe guardar:

- clientes registrados
- productos disponibles
- ventas realizadas
- detalle de productos vendidos en cada venta

La empresa quiere comenzar a responder preguntas reales como:

- ¿qué clientes compran más?
- ¿qué productos se venden más?
- ¿cuántas ventas se realizan?
- ¿qué ventas tienen más de un producto?
- ¿qué clientes han comprado varias veces?

# ❓Preguntas de negocio que deben resolver en `report.sql`

## 🔹 Nivel 1 — Consultas básicas

1. Mostrar todos los clientes registrados.
2. Mostrar todos los productos disponibles.
3. Mostrar todas las ventas realizadas.
4. Mostrar solo el nombre y email de los clientes.
5. Mostrar solo el nombre y precio de los productos.

---

## 🔹 Nivel 2 — Filtros y orden

1. Mostrar los productos cuyo precio sea mayor a `50000`.
2. Mostrar las ventas realizadas en la fecha `2026-04-02`.
3. Mostrar los productos ordenados de mayor a menor precio.
4. Mostrar los clientes ordenados alfabéticamente por nombre.
5. Mostrar los detalles de venta cuya cantidad sea mayor o igual a `2`.

---

## 🔹 Nivel 3 — Agregaciones

1. ¿Cuántos clientes existen en total?
2. ¿Cuántos productos existen en total?
3. ¿Cuántas ventas existen en total?
4. ¿Cuál es el precio promedio de los productos?
5. ¿Cuál es la suma total de precios de todos los productos?

---

## 🔹 Nivel 4 — JOIN

1. Mostrar cada venta con:
- nombre del cliente
- fecha de la venta
1. Mostrar el detalle de ventas con:
- id de la venta
- nombre del producto
- cantidad
1. Mostrar:
- nombre del cliente
- id de la venta
- fecha
1. Mostrar:
- nombre del producto
- cantidad vendida
- id de la venta

---

## 🔹 Nivel 5 — GROUP BY + HAVING

1. Mostrar cuántas ventas ha realizado cada cliente.
2. Mostrar solo los clientes que han realizado más de una venta.
3. Mostrar cuántas veces aparece cada producto en `detalle_venta`.
4. Mostrar solo los productos que aparecen más de una vez en el detalle de ventas.
5. Mostrar las ventas que tienen más de un producto asociado.
6. Mostrar los clientes cuya suma total de unidades compradas sea mayor a `2`.

---

## 🔹 Nivel 6 — Consulta trampa

1. Crear una consulta correctamente escrita que no devuelva resultados.

Luego deben agregar un comentario SQL explicando por qué el resultado vacío es correcto.

# 🐘 schema.sql

```sql
DROP TABLE IF EXISTS detalle_venta;
DROP TABLE IF EXISTS ventas;
DROP TABLE IF EXISTS productos;
DROP TABLE IF EXISTS clientes;

CREATE TABLE clientes (
    id_cliente INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE
);

CREATE TABLE productos (
    id_producto INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    precio NUMERIC(10,2) NOT NULL CHECK (precio > 0)
);

CREATE TABLE ventas (
    id_venta INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    fecha DATE NOT NULL,
    id_cliente INT NOT NULL,
    CONSTRAINT fk_ventas_cliente
        FOREIGN KEY (id_cliente)
        REFERENCES clientes(id_cliente)
        ON DELETE RESTRICT
        ON UPDATE CASCADE
);

CREATE TABLE detalle_venta (
    id_detalle INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_venta INT NOT NULL,
    id_producto INT NOT NULL,
    cantidad INT NOT NULL CHECK (cantidad > 0),
    CONSTRAINT fk_detalle_venta_venta
        FOREIGN KEY (id_venta)
        REFERENCES ventas(id_venta)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_detalle_venta_producto
        FOREIGN KEY (id_producto)
        REFERENCES productos(id_producto)
        ON DELETE RESTRICT
        ON UPDATE CASCADE
);
```

# 🧑‍🏫 Notas

Este challenger está bueno porque permite evaluar progresivamente:

- primero lectura de tablas
- luego filtros
- después joins
- finalmente pensamiento analítico con `GROUP BY` y `HAVING`

Además, el `seed.sql` quedó armado para que sí existan resultados interesantes en:

- clientes con más de una venta
- productos repetidos en varias ventas
- ventas con más de un producto
- clientes con más de 2 unidades compradas
