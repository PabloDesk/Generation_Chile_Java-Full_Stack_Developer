# Sistema de Gestión Clínica

> [!NOTE]
> ## *Clínica VidaPlus*
>
> **Caso práctico completo – PostgreSQL**

---

## 📋 CONTEXTO DEL PROBLEMA

La **Clínica VidaPlus** es un centro de salud privado que atiende consultas médicas ambulatorias.

Actualmente, la información de pacientes, médicos y atenciones se gestiona de forma dispersa, lo que dificulta:

- Controlar la atención médica
- Obtener estadísticas clínicas
- Analizar rendimiento médico
- Detectar pacientes sin atención

La gerencia solicita diseñar una **base de datos relacional normalizada** que permita **gestionar, consultar y analizar** la información clínica.

---

# 🧠 REGLAS DE NEGOCIO

### 👤 PACIENTES

- Cada paciente tiene un **RUT único**
- Puede tener **muchas consultas**
- Puede no haber tenido consultas aún
- Tiene estado: `activo`, `inactivo`

### 👨‍⚕️ MÉDICOS

- Cada médico tiene RUT único
- Pertenece a **una especialidad**
- Puede atender **muchas consultas**
- Puede no tener consultas registradas

### 🧬 ESPECIALIDADES

- Cardiología, Pediatría, Traumatología, etc.
- Una especialidad puede tener **muchos médicos**

### 🩺 CONSULTAS

- Relaciona paciente + médico
- Tiene fecha, costo y observación
- Pertenece a un **período clínico**
- Puede existir consulta **sin diagnóstico aún**

### 📄 DIAGNÓSTICOS

- Se asocian a una consulta
- Tienen descripción y nivel de gravedad
- No todas las consultas tienen diagnóstico

### 📆 PERÍODOS

- Año + semestre (1 o 2)
- Todas las consultas pertenecen a un período

# 🧱 schema.sql

## (Creación de tablas – PostgreSQL)

```sql
CREATE DATABASE clinica_vidaplus;
```

> [!NOTE]
> Conectarse a la base clinica_vidaplus

```sql
-- Especialidades
CREATE TABLE especialidades (
    id_especialidad INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL UNIQUE
);

-- Médicos
CREATE TABLE medicos (
    id_medico INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    rut VARCHAR(12) UNIQUE NOT NULL,
    nombre VARCHAR(100) NOT NULL,
    id_especialidad INT NOT NULL,
    estado VARCHAR(20) NOT NULL,
    FOREIGN KEY (id_especialidad) REFERENCES especialidades(id_especialidad)
);

-- Pacientes
CREATE TABLE pacientes (
    id_paciente INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    rut VARCHAR(12) UNIQUE NOT NULL,
    nombre VARCHAR(100) NOT NULL,
    fecha_nacimiento DATE NOT NULL,
    estado VARCHAR(20) NOT NULL
);

-- Períodos clínicos
CREATE TABLE periodos (
    id_periodo INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    anio INT NOT NULL,
    semestre INT CHECK (semestre IN (1,2))
);

-- Consultas médicas
CREATE TABLE consultas (
    id_consulta INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_paciente INT NOT NULL,
    id_medico INT NOT NULL,
    id_periodo INT NOT NULL,
    fecha DATE NOT NULL,
    costo INT CHECK (costo >= 0),
    observacion TEXT,
    FOREIGN KEY (id_paciente) REFERENCES pacientes(id_paciente),
    FOREIGN KEY (id_medico) REFERENCES medicos(id_medico),
    FOREIGN KEY (id_periodo) REFERENCES periodos(id_periodo)
);

-- Diagnósticos
CREATE TABLE diagnosticos (
    id_diagnostico INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_consulta INT UNIQUE,
    descripcion TEXT NOT NULL,
    gravedad VARCHAR(20),
    FOREIGN KEY (id_consulta) REFERENCES consultas(id_consulta)
);

```

# 📊 report.sql

## (Consultas – Solución del caso)

---

### 1️⃣ Pacientes con consultas registradas (INNER JOIN)

```sql
SELECT p.nombre, c.fecha
FROM pacientes p
INNER JOIN consultas c
  ON p.id_paciente = c.id_paciente;
```

---

### 2️⃣ Todos los pacientes y sus consultas (LEFT JOIN)

```sql
SELECT p.nombre, c.fecha
FROM pacientes p
LEFT JOIN consultas c
  ON p.id_paciente = c.id_paciente;
```

---

### 3️⃣ Consultas sin diagnóstico

```sql
SELECT c.id_consulta, c.fecha
FROM consultas c
LEFT JOIN diagnosticos d
  ON c.id_consulta = d.id_consulta
WHERE d.id_diagnostico IS NULL;
```

---

### 4️⃣ Cantidad de consultas por médico

```sql
SELECT m.nombre, COUNT(c.id_consulta) AS total_consultas
FROM medicos m
LEFT JOIN consultas c
  ON m.id_medico = c.id_medico
GROUP BY m.nombre;
```

---

### 5️⃣ Promedio de costo por especialidad

```sql
SELECT e.nombre, AVG(c.costo) AS promedio
FROM especialidades e
JOIN medicos m ON e.id_especialidad = m.id_especialidad
JOIN consultas c ON m.id_medico = c.id_medico
GROUP BY e.nombre;
```

---

### 6️⃣ Médicos sin consultas

```sql
SELECT m.nombre
FROM medicos m
LEFT JOIN consultas c
  ON m.id_medico = c.id_medico
WHERE c.id_consulta IS NULL;
```

---

### 7️⃣ Pacientes atendidos en segundo semestre

```sql
SELECT DISTINCT p.nombre
FROM pacientes p
JOIN consultas c ON p.id_paciente = c.id_paciente
JOIN periodos pe ON c.id_periodo = pe.id_periodo
WHERE pe.semestre = 2;
```

---

### 8️⃣ Diagnósticos de gravedad media o alta

```sql
SELECT d.descripcion, d.gravedad
FROM diagnosticos d
WHERE d.gravedad IN ('media','alta');
```

---

### 9️⃣ Total facturado por período

```sql
SELECT pe.anio, pe.semestre, SUM(c.costo) AS total
FROM consultas c
JOIN periodos pe ON c.id_periodo = pe.id_periodo
GROUP BY pe.anio, pe.semestre;
```

---

### 🔟 Pacientes sin consultas registradas

```sql
SELECT p.nombre
FROM pacientes p
LEFT JOIN consultas c
  ON p.id_paciente = c.id_paciente
WHERE c.id_consulta IS NULL;
```
