# SQL – WHERE y HAVING

## 🎯 Objetivo de esta página

Comprender **qué hacen WHERE y HAVING**, en qué se diferencian y **cuándo usar cada uno**, para filtrar datos de forma correcta en SQL.

---

> [!IMPORTANT]
> ## 🧠 Idea clave inicial
> 
> WHERE filtra filas antes de agrupar
> 
> **HAVING filtra grupos después de agrupar**


Aunque ambos se usan para filtrar, **no funcionan igual** ni se ejecutan en el mismo momento.

---

## 🧩 ¿Cómo procesa SQL una consulta?

De forma simplificada, SQL ejecuta una consulta en el siguiente orden:

1. `FROM`
2. `WHERE`
3. `GROUP BY`
4. `HAVING`
5. `SELECT`
6. `ORDER BY`

📌 Este orden explica por qué:

- `WHERE` no puede usar funciones de agregación
- `HAVING` sí puede hacerlo

---

## 🔹 WHERE – Filtrar filas individuales

### ¿Qué hace WHERE?

`WHERE` filtra **registros individuales** antes de que se realicen agrupaciones.

Se utiliza cuando se quiere:

- Buscar datos específicos
- Filtrar por valores
- Trabajar con columnas normales

---

### 📌 Ejemplo básico

```sql
SELECT *
FROM productos
WHERE precio > 20000;
```

👉 Se evalúa producto por producto.

---

### 📌 WHERE con texto

```sql
SELECT *
FROM clientes
WHERE nombre = 'Ana Pérez';
```

---

### 📌 WHERE con fechas

```sql
SELECT *
FROM ventas
WHERE fecha = '2024-09-01';
```

---

### 📌 WHERE con múltiples condiciones

```sql
SELECT *
FROM productos
WHERE precio > 20000
  AND precio < 200000;
```

---

### ❌ Error común con WHERE

```sql
SELECT COUNT(*)
FROM productos
WHERE COUNT(*) > 2;
```

⛔ Incorrecto

Las funciones de agregación (`COUNT`, `AVG`, etc.) **no existen aún** en la etapa WHERE.

---

## 🔸 HAVING – Filtrar grupos de datos

### ¿Qué hace HAVING?

`HAVING` filtra **resultados agrupados**, es decir, trabaja **después de un `GROUP BY`**.

Se usa cuando se quiere:

- Filtrar resultados de agregaciones
- Aplicar condiciones sobre `COUNT`, `AVG`, `SUM`, etc.

---

### 📌 Ejemplo con GROUP BY

```sql
SELECT id_cliente, COUNT(*) AS total_compras
FROM ventas
GROUP BY id_cliente;

```

👉 Agrupa las compras por cliente.

---

### 📌 Filtrar grupos con HAVING

```sql
SELECT id_cliente, COUNT(*) AS total_compras
FROM ventas
GROUP BY id_cliente
HAVING COUNT(*) > 1;
```

👉 Solo muestra clientes con más de una compra.

---

### ❌ Error común con HAVING

```sql
SELECT *
FROM ventas
HAVING fecha = '2024-09-01';
```

⛔ Incorrecto

`HAVING` **no reemplaza** a `WHERE` y no se usa para filtrar filas sin agrupación.

---

## 🧠 WHERE y HAVING juntos

Ambos se pueden usar en la **misma consulta**, cada uno en su rol.

---

### 📌 Ejemplo combinado

```sql
SELECT id_cliente, COUNT(*) AS total_compras
FROM ventas
WHERE fecha >= '2024-09-01'
GROUP BY id_cliente
HAVING COUNT(*) >= 2;
```

👉 `WHERE` filtra las compras

👉 `GROUP BY` agrupa

👉 `HAVING` filtra los grupos

---

## 🧠 Comparación clara

| WHERE | HAVING |
| --- | --- |
| Filtra filas | Filtra grupos |
| Se ejecuta antes del GROUP BY | Se ejecuta después |
| No usa agregaciones | Usa agregaciones |
| Muy común | Menos frecuente |

---

## 🧠 ¿Cuál usar?

- **Usa WHERE** cuando trabajas con datos “normales”
- **Usa HAVING** cuando trabajas con resultados de `COUNT`, `AVG`, `SUM`, etc.
- **No son intercambiables**

---

## 🧠 Idea clave para cerrar

> [!IMPORTANT]
> WHERE decide qué datos entran al análisis
> 
> HAVING decide **qué resultados se muestran**

- 🐬 **SCRIPT SQL — MySQL 8**
    
    ## 1️⃣ Crear base de datos
    
    ```sql
    CREATE DATABASE IF NOT EXISTS filtros_sql;
    USE filtros_sql;
    ```
    
    ---
    
    ## 2️⃣ Tabla clientes
    
    ```sql
    CREATE TABLE clientes (
        id_cliente INT PRIMARY KEY AUTO_INCREMENT,
        nombre VARCHAR(100) NOT NULL,
        email VARCHAR(100) NOT NULL,
        activo BOOLEAN NOT NULL
    );
    ```
    
    ---
    
    ## 3️⃣ Tabla productos
    
    ```sql
    CREATE TABLE productos (
        id_producto INT PRIMARY KEY AUTO_INCREMENT,
        nombre VARCHAR(100) NOT NULL,
        precio INT NOT NULL CHECK (precio > 0),
        stock INT NOT NULL CHECK (stock >= 0)
    );
    ```
    
    ---
    
    ## 4️⃣ Tabla ventas
    
    ```sql
    CREATE TABLE ventas (
        id_venta INT PRIMARY KEY AUTO_INCREMENT,
        id_cliente INT NOT NULL,
        fecha DATE NOT NULL,
        total INT NOT NULL CHECK (total >= 0),
        FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente)
    );
    ```
    
    ---
    
    ## 5️⃣ Tabla detalle_venta
    
    ```sql
    CREATE TABLE detalle_venta (
        id_detalle INT PRIMARY KEY AUTO_INCREMENT,
        id_venta INT NOT NULL,
        id_producto INT NOT NULL,
        cantidad INT NOT NULL CHECK (cantidad > 0),
        FOREIGN KEY (id_venta) REFERENCES ventas(id_venta),
        FOREIGN KEY (id_producto) REFERENCES productos(id_producto)
    );
    ```

    CREATE TABLE productos (
